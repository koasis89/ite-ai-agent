# 평가 구조와 제안 전략

## 1. 협상에 의한 계약의 구조

공제조합·보험 유관기관의 SI 발주는 대부분 「국가계약법」·「소프트웨어 진흥법」 체계를 준용한 **협상에 의한 계약**이다. 구조를 이해해야 전략이 선다.

- 종합점수 = 기술능력평가(통상 90%) + 입찰가격평가(통상 10%)
- 기술점수가 배점한도의 **85% 미만이면 협상대상에서 제외**된다 — 가격을 아무리 낮춰도 소용없다
- 종합점수 1위가 우선협상대상자가 되고, 협상이 성립하면 차순위와는 협상하지 않는다 — 사실상 기술평가 1위가 수주한다
- **차등점수제**: 발주기관이 순위 간 점수차(통상 2~3점)를 지정하면, 원점수 차이가 근소해도 최종점수는 강제로 벌어진다. "적당히 잘 쓴 2등"의 가치가 없다는 뜻이다. 1위를 목표로 쓴다

가격평가는 기획재정부 계약예규 산식을 따르며 배점이 작아 변별력이 낮다. 승부는 기술평가에서 난다.

## 2. 배점표 기반 작성 역량 배분

RFP의 기술평가 배점표를 놓고 다음을 수행한다:

1. **배점 상위 항목을 식별한다.** 통상 기능요구사항(5~6점), 사업이해도(5점), 추진전략(5점), 데이터요구사항(5점), 시스템운영(5점), 관리방법론(5점)이 크고, 테스트·품질·교육 등은 2점 내외다.
2. **장표 수와 완성도를 배점에 비례시킨다.** 2점짜리 항목에 10장을 쓰고 6점짜리에 5장을 쓰는 실수를 하지 않는다. 배점이 작은 항목은 표준적·모범적으로 짧게, 큰 항목은 구체적 방안·다이어그램·사례로 깊게 쓴다.
3. **평가기준 문장을 답안 채점 기준으로 삼는다.** 배점표의 평가기준(예: "일정 및 위험요소를 고려하여 얼마나 타당한 추진전략을 수립하였는지")에 등장하는 명사를 장표에 그대로 등장시킨다. 위 예라면 추진전략 파트에 반드시 '일정 리스크'와 '위험요소별 대응'이 보여야 한다. 평가위원은 평가기준 문장을 들고 제안서에서 근거를 찾는다 — 찾기 쉽게 만들어 주는 것이 점수다.

## 3. 차별화 포인트 도출 프레임

배점 상위 항목마다 차별화 포인트를 1~2개 정의한 후 본문을 쓴다. 도출 순서:

1. **RFP 반복 키워드에 응답한다.** 발주기관이 배경·기대효과에서 반복한 키워드(AI 자동화, 클라우드, 조합원 서비스, 유연한 상품 구조 등)마다 "우리는 이것을 이렇게 실현한다"를 만든다.
2. **발주기관의 숨은 불안을 해소한다.** 공제조합은 IT 조직이 작다. 따라서 "구축 후 우리끼리 운영할 수 있는가", "기존 데이터가 안전하게 넘어오는가", "현업이 쓰기 쉬운가"가 실질적 불안이다. 데이터 마이그레이션 검증 체계, 안정화 지원, 교육·기술이전을 구체화하면 배점 대비 큰 신뢰를 얻는다.
3. **도메인 이해를 증명한다.** 공제 업무 용어와 프로세스(가입설계→청약→계약→사건→심사→보상→소송→회계)를 정확히 구사하고, 현행 시스템의 문제(RFP 현황 절 참조)를 To-Be 방안과 1:1로 연결한다.
4. **검증 가능한 자산을 제시한다.** 유사 실적, 보유 솔루션·프레임워크, 인증(SP·CMMI·ISO), 프로토타입 시연 계획 등. 제안사 정보가 없으면 플레이스홀더로 두고 사용자에게 요청한다.

## 4. 감점·탈락 리스크 통제

- **애매모호 표현**("~할 수 있다", "고려하고 있다", "검토하겠다")은 평가 시 불가능으로 처리된다. scripts/check_expressions.py로 전수 검사한다.
- **요구사항 누락**: 수용표에서 하나라도 빠지면 미충족 처리 근거가 된다. 전 ID 커버리지를 검증한다.
- **목차 이탈**: 지정 목차의 순서·항목 누락은 "지침 미준수"로 읽힌다.
- **허위·과장 기재**: 실적·인력의 허위 기재는 탈락 사유다. 확인되지 않은 정보는 절대 채워 넣지 않는다.
- **PM 요건**: PM은 공고일 전부터 재직자여야 하고 제안설명회에서 직접 발표해야 하는 것이 일반적이다. 인력 구성 제안 시 이 제약을 반영한다.
- **하도급 구조**: SW진흥법상 하도급 50% 초과 금지, 10% 초과 시 공동수급 구성 의무 등을 위반하는 구성을 제안하지 않는다. 하도급이 없으면 상생협력·하도급 항목에서 통상 최고 등급을 받으므로, 구성이 자유롭다면 "하도급 없음 + 중소기업 단독/주도"가 배점상 유리하다는 점을 사용자에게 알린다.

## 5. 전략 보고서 산출 형식

전략 컨설팅을 요청받으면 다음 구조로 정리한다:

```
# [사업명] 제안 전략
## 1. 승부처 요약 (배점 상위 항목과 차등점수제 등 게임의 규칙)
## 2. 발주기관 우선순위 분석 (반복 키워드, 숨은 불안)
## 3. 항목별 전략 매트릭스 (평가항목 | 배점 | 평가기준 핵심어 | 우리의 차별화 포인트 | 필요한 근거 자료)
## 4. 리스크와 통제 방안 (감점 요인, 독소조항, 참가자격)
## 5. 제안사에 요청할 정보 목록
```
