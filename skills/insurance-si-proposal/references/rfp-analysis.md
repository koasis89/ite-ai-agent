# RFP 분석 가이드

RFP를 받으면 본문 작성 전에 아래 항목을 모두 추출하여 분석 노트로 정리한다. 이 노트가 이후 모든 산출물의 단일 기준(single source of truth)이 된다.

## 1. 추출 체크리스트

### 사업 개요
- 사업명, 발주기관(주관기관), 사업기간, 사업비(VAT 포함 여부), 계약방식(협상에 의한 계약 / 적격심사 등)
- 사업 범위: 세부 사업 단위로 분해한다 (예: 홈페이지 개선 / 모바일 앱 / 코어시스템 고도화 / 인프라 전환)
- 추진배경·필요성: 발주기관이 스스로 밝힌 문제의식 — 사업이해도 파트의 핵심 소재

### 요구사항 목록
- 요구사항 분류 체계와 ID 전체 목록을 표로 추출한다. 공공 SI RFP는 통상 다음 접두어 체계를 쓴다:
  - SFR(기능), ECR(시스템 장비구성), PER(성능), INR(인터페이스), DAR(데이터), TER(테스트), SER(보안), QUR(품질), COR(제약사항), PMR(프로젝트 관리), PSR(프로젝트 지원)
- 각 요구사항의 고유번호·명칭·세부내용 요지를 추출한다. 기능요구사항(SFR)은 개수가 많으므로 업무 영역별로 그룹핑한다 (예: 가입·계약 / 사건·조사 / 심사 / 소송 / 보상·회계 / 기준정보·운영 / AI·통계 / 채널)
- "관련 요구사항" 상호참조가 있으면 기록한다 — 구현방안의 연계 논리에 쓰인다

### 제안서 목차와 작성지침
- RFP가 지정한 제안서 목차를 그대로 옮겨 적는다. 이 목차는 변경 불가이며 세분화만 허용되는 것이 일반적이다
- "제안서 세부 작성지침" 표(항목별 작성 방법)를 추출한다 — 각 파트에 무엇을 쓰라고 요구하는지가 여기 명시된다
- 필수 제출물 확인: 요구사항 수용표(수용여부 참조표), 기술평가항목 위치표(인덱스표), 제안요약서, 각종 서식
- 작성 금지·유의사항: 애매모호 표현 금지, 한글 작성 원칙, 약어 풀어쓰기, 허위 기재 시 탈락 등

### 평가 기준
- 종합평가 구성비 (통상 기술 80~90% + 가격 10~20%)
- 기술평가 배점표 전체: 평가부문 — 세부평가항목 — 평가기준 — 배점. 이 표가 전략 수립의 근거이므로 반드시 원문 그대로 추출한다
- 협상적격 기준(통상 기술점수 배점한도의 85% 이상), 차등점수제 적용 여부와 점수차
- 가점·감점 항목 (SP인증 우대, 중소기업 참여비율, 하도급 적정성 등)

### 참가자격·법적 제약
- 입찰 참가자격 (직접생산확인증명, 소프트웨어사업자 신고, 상호출자제한기업집단 배제 등)
- 하도급 제한 (SW진흥법: 하도급 50% 초과 금지, 재하도급 원칙 불허, 10% 초과 시 공동수급 구성)
- 공동수급 조건 (구성원 수 제한, 최소지분율)
- PM 요건 (공고일 이전부터 평가일까지 재직, PM 직접 발표 등)

### 일정·절차
- 제안서 제출 기한·방법, 제안요청 설명회, 제안설명회(PT) 방식
- 협상 절차와 가격협상 기준금액 산정 방식

## 2. 분석 노트 산출 형식

추출이 끝나면 다음 구조의 분석 노트(md)를 만든다:

```
# [사업명] RFP 분석
## 1. 사업 스냅샷 (개요 표)
## 2. 사업 범위 분해
## 3. 요구사항 인벤토리 (ID 전체 표 + 영역별 그룹핑)
## 4. 지정 목차 및 파트별 작성지침 요약
## 5. 평가 배점표 (원문 그대로)
## 6. 제약·리스크 (참가자격, 하도급, 금지사항, 독소조항)
## 7. 발주기관이 강조하는 것 (배경·기대효과에서 반복되는 키워드)
```

7번이 중요하다. RFP에서 반복 등장하는 키워드(예: AI 자동화, 클라우드 전환, 조합원 서비스, 유연한 상품 구조)는 발주기관의 우선순위 신호이며, 제안서의 추진전략·차별화 포인트는 이 키워드에 정면으로 응답해야 한다.

## 3. 독소조항·리스크 스캔

다음 유형의 조항을 발견하면 분석 노트의 리스크 절에 기록하고, 전략 컨설팅 요청 시 사용자에게 알린다:

- "명기되지 않은 사항도 필요 시 사업내용에 반영, 입찰금액에 포함 간주" — 범위 무한 확장 리스크
- 지식재산권 발주기관 귀속, 제안서의 계약조건 편입
- 하자담보 기간, 무상 유지보수 요구
- 상주 개발, 보안 요건에 따른 비용 요소
- 참여인력 임의 교체 불가 조항

이는 제안 여부·가격 산정 판단에 영향을 주는 정보이므로 누락하지 않는다.
