# 요구사항 수용표 · 기술평가항목 위치표

두 표 모두 RFP가 제출을 의무화하는 것이 일반적이다. xlsx로 생성한다 (xlsx 스킬 병용).

## 1. 요구사항 수용표 (제안요청서 수용여부 참조표)

RFP의 **모든 요구사항 ID**를 빠짐없이 행으로 나열한다. 하나라도 누락되면 미충족 처리 근거가 된다.

| 컬럼 | 내용 | 규칙 |
|---|---|---|
| 분류 | 요구사항 대분류 (기능/성능/보안 등) | RFP 분류 체계 그대로 |
| 요구사항 번호 | SFR-001 등 고유번호 | RFP 표기 그대로, 순서 유지 |
| 요구사항 명칭 | RFP의 명칭 | 임의 수정 금지 |
| 수용여부 | 수용 / 부분수용 / 미수용 | 원칙적으로 전 항목 "수용". 부분수용·미수용은 탈락 리스크이므로 사용자와 반드시 협의 |
| 제안서 위치 | 제안서 목차·페이지 (예: Ⅲ-1, p.45~48) | 실제 본문 위치와 일치해야 함 |
| 이행방안 요약 | 1~2문장 | 단정형으로. 비워두지 않는다 |

작성 절차:
1. RFP 분석 노트의 요구사항 인벤토리에서 전 ID를 가져온다
2. 제안서 본문(또는 목차 계획)과 매핑한다 — 본문이 아직 없으면 목차 수준으로 위치를 지정하고, 본문 완성 후 페이지를 갱신한다
3. 매핑되지 않은 ID가 있으면 본문에 해당 내용을 보강한다 (표를 본문에 맞추는 게 아니라 본문을 요구사항에 맞춘다)
4. 완성 후 커버리지를 검증한다: RFP ID 집합 = 수용표 ID 집합 (스크립트 또는 대조로 확인)

## 2. 기술평가항목 위치표 (인덱스표)

평가위원이 배점표를 들고 채점할 때 쓰는 안내표다. 기술평가 배점표의 **세부평가항목**별로 제안서·제안요약서의 위치를 매핑한다.

| 컬럼 | 내용 |
|---|---|
| 평가부문 | 전략및방법론 / 기술및기능 / … |
| 세부평가항목 | 사업이해도, 추진전략, … (배점표 그대로) |
| 배점 | 배점표 그대로 |
| 제안서 위치 | 목차 번호 + 페이지 범위 |
| 제안요약서 위치 | 요약서 페이지 |

이 표를 만들다가 특정 평가항목에 대응하는 본문이 얇다는 것이 드러나면, 그것이 곧 감점 예상 지점이다 — 본문 보강 신호로 사용자에게 알린다.

## 3. 산출 형식 (xlsx)

- 시트 1: 요구사항 수용표, 시트 2: 기술평가항목 위치표
- 머리행 고정, 분류별 필터 가능하게, 분류 경계에 시각 구분(배경색)
- 인쇄를 고려해 A4 가로 기준 열 너비 조정
