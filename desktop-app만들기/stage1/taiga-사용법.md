## 2. 사전 준비

진행 전 아래 항목을 준비한다.

1. 스프린트 기간
- 시작일, 종료일
- 예: 2026-05-27 ~ 2026-06-10

2. 스프린트 목표(Goal)
- 이번 스프린트에서 반드시 달성해야 하는 1~2문장

3. 백로그 후보 목록
- 제목
- 설명
- 우선순위
- 예상 포인트(Story Point)
- 담당자(있으면)

4. 팀 용량(Capacity)
- 팀원별 가용일
- 휴가/회의 등 비가용 시간 반영

- [ ] 기존 항목 발견 시 신규 생성 대신 기존 항목에 Task/코멘트/링크로 연동

## 4. 백로그(User Story) 등록

0. 기존 항목 검색(중복 방지)
- Backlog 검색창에 식별자/제목 키워드 입력
- 이미 존재하면 생성하지 말고 해당 Story를 재사용

1. 프로젝트 좌측 메뉴에서 Backlog 이동

2. User Story 생성
- New User Story 또는 + Add 버튼 클릭

3. 필수값 입력
- Subject(제목): 작업 한 줄 요약
- Description(설명): 완료 기준/요구사항/제약사항

4. 권장값 입력
- Points: 스토리 포인트
- Priority: High/Normal/Low
- Tags: 기능영역 태그
- Assignee: 담당자

5. 저장
- Create/Save 클릭

6. 반복
- 스프린트 후보 항목들을 같은 방식으로 연속 등록

7. ID 매핑 갱신
- 생성 또는 재사용한 Story ID를 `change-winapp-실무등록용-백로그.md` 0번 섹션에 즉시 반영

작성 템플릿(권장):
- 제목: [영역] 작업 요약
- 설명:
  - 배경:
  - 요구사항:
  - 완료조건(DoD):
  - 비고:

검증 기준:
- Backlog 리스트에 생성한 User Story가 보이고,
- 각 항목의 포인트/우선순위가 누락되지 않았는지 확인

---

## 5. 스프린트(Milestone) 생성

1. Backlog 화면에서 Sprint/Milestone 생성 버튼 클릭
- 보통 "Create Sprint" 또는 "New Milestone"

2. 스프린트 기본 정보 입력
- Name: 예) Sprint 1
- Start date: 시작일
- End date: 종료일
- Goal(또는 Description): 스프린트 목표

3. 저장
- Create/Save

검증 기준:
- 스프린트 섹션이 생성되고 기간이 올바르게 표시되어야 함

---

## 6. 백로그를 스프린트에 배정

1. Backlog의 User Story 선택

2. 스프린트로 이동
- Drag & Drop 또는
- 항목 편집에서 Milestone 필드에 해당 Sprint 선택

3. 우선순위 재정렬
- 스프린트 내 실행 순서 기준으로 정렬

4. 용량 점검
- 총 포인트가 팀 용량 범위를 초과하지 않도록 조정

검증 기준:
- 스프린트에 필요한 User Story가 모두 포함됨
- 과부하(포인트 초과) 여부 확인 완료

---

## 7. 태스크(Task)로 세분화 (권장)

1. 각 User Story 상세 화면 진입

2. Task 생성
- 구현/테스트/문서화 등 실행 단위로 분해

3. 담당자 및 상태 설정
- Assignee 지정
- 초기 상태(New/Open)

검증 기준:
- User Story마다 최소 1개 이상의 실행 Task 존재

---

## 8. 스프린트 시작 전 점검 체크리스트

- [ ] 스프린트 기간(시작/종료일) 확정
- [ ] 스프린트 Goal 입력
- [ ] 필수 User Story 모두 등록
- [ ] Story Point 입력 완료
- [ ] 담당자 배정 완료(또는 미배정 정책 명시)
- [ ] 총 포인트가 팀 용량 이내
- [ ] 중요 User Story에 Task 분해 완료

---

## 9. 운영 규칙(권장)

1. 상태 업데이트 규칙
- Daily Scrum 전에 각자 Task 상태 업데이트

2. 변경 관리 규칙
- 스프린트 중 신규 항목 추가는 PO 승인 후 반영

3. 완료 기준(DoD)
- 코드/테스트/문서 반영 + 리뷰 완료 시 Done 처리

4. 회고 데이터 축적
- Sprint 종료 시 미완료 항목 원인 기록

---

## 10. 문제 발생 시 점검 포인트

1. 프로젝트가 보이지 않는 경우
- 권한(Role) 확인
- 올바른 조직/워크스페이스 로그인 여부 확인

2. Sprint 생성 버튼이 없는 경우
- 프로젝트 권한이 읽기 전용인지 확인
- 관리자 계정으로 권한 재부여

3. Story 이동이 안 되는 경우
- Milestone 선택 방식(드래그/편집) 둘 다 시도
- 브라우저 캐시/새로고침 후 재시도

---

## 11. 빠른 실행 요약 (5분 버전)

1. 로그인 -> AI-Isaki 프로젝트 선택
2. Backlog에서 User Story 일괄 생성
3. Sprint 1 생성(기간+Goal)
4. User Story를 Sprint 1로 배정
5. 총 포인트/담당자 확인 후 확정

이 5단계를 완료하면 스크럼 스프린트 운영 준비가 끝난다.
