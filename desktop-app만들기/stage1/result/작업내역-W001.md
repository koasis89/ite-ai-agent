# 작업내역-W001

## 1. 작업 식별 정보
- 작업 ID: WIN-001
- 작업명: 실행 코어 계약(Command Contract) 정의
- 기준 문서: change-winapp-phase1-tickets.md
- 작업 일자: 2026-05-22
- 작업 브랜치: webapp (직전 브랜치 전환 작업 기준)

## 2. 작업 목표
WIN-001의 완료 기준에 맞춰 CLI와 Desktop Main에서 공통으로 사용할 실행 계약 타입을 신규 정의한다.

핵심 요구사항:
- CommandRequest 정의
- CommandResult 정의
- CommandEvent 정의
- 런타임 구분 필드(runtime: cli | desktop-main)
- 공통 에러 코드 enum 정의
- 타입 파일 빌드/타입 검증 가능 상태 확보

## 3. 실제 변경 내역
### 3.1 신규 디렉터리 생성
- src/core/contracts

### 3.2 신규 파일 생성
- src/core/contracts/command.ts

### 3.3 구현 상세
아래 항목을 신규 파일에 반영했다.

1) 런타임 계약
- COMMAND_RUNTIMES 상수 정의
- CommandRuntime 타입 정의
- 허용 런타임: cli, desktop-main

2) 에러 코드 계약
- CommandErrorCode enum 정의
- 포함 코드:
  - INVALID_REQUEST
  - UNKNOWN_COMMAND
  - VALIDATION_FAILED
  - COMMAND_FAILED
  - PERMISSION_DENIED
  - INTERNAL_ERROR

3) 요청/응답 계약
- CommandRequest 인터페이스 정의
  - command
  - args
  - runtime
  - correlationId
  - context
- CommandResult 인터페이스 정의
  - ok
  - code
  - message
  - data
  - logs
  - diagnostics

4) 이벤트 계약
- CommandEventName 유니온 정의
  - command.started
  - command.progress
  - command.completed
  - command.failed
- 이벤트 공통 베이스 타입 정의
  - name
  - command
  - runtime
  - correlationId
  - timestamp
- 세부 이벤트 타입 정의
  - CommandStartedEvent
  - CommandProgressEvent
  - CommandCompletedEvent
  - CommandFailedEvent
- CommandEvent 유니온 타입 정의

5) 사용 예시
- 파일 하단에 CommandRequest 예시 주석 추가

1) 전체 빌드 블로커 존재
- 현재 저장소에는 WIN-001 변경 이전부터 존재한 타입 오류가 있어 전체 빌드가 실패한다.
