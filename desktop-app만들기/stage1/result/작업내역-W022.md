## 2. 목표

- WIN-021 의 분기 패턴 위에서 **인자 검증이 필요한** 4개 명령을 추가한다.
- 명령별 zod 스키마를 도입해 인자 위반은 `INVALID_REQUEST` 로, 환경 부재(예: sender 없음)는 `COMMAND_FAILED` 로 명확히 구분.
- WIN-023 의 외부 프로세스 호출 전, **모달 호출(`question_ask`) / 타이밍(`noop_sleep`) / 왕복(`noop_echo`) / 필드 선택(`state_get_field`)** 능력을 노출해 다음 그룹의 의존을 확보.

## 3. 변경 내역

### 변경 파일

- [desktop/ipc/commands.ts](../../../desktop/ipc/commands.ts)

## 4. 신규 명령 요약

| 명령 | 인자 | 검증 규약 | 응답 핵심 |
|---|---|---|---|
| `state_get_field` | `args[0]=field` | enum(platform/runtime/versions/timestamp/status/command) | `{ field, value }` |
| `question_ask` | `args[0]=question`(1~200), `args[1..5]=options`(1~40, `[\p{L}\p{N} _-]`) | zod tuple-like array + 유니코드 정규식 | `{ question, answer }` |
| `noop_echo` | `args[]` | 길이 ≤ 10, 각 ≤ 200자 | `{ args, count }` |
| `noop_sleep` | `args[0]=ms` | 정수 0~5000 | `{ requestedMs, actualMs }` |
