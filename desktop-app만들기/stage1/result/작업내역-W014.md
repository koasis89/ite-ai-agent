# 작업내역-W014

## 1. 작업 식별 정보
- 작업 ID: WIN-014
- 작업명: 운영형 UI 전환 — 명령 히스토리/검색/필터
- 기준 문서: change-winapp-phase2-tickets.md (WIN-014 섹션)
- 작업 일자: 2026-05-23
- 작업 브랜치: webapp

## 2. 작업 목표
진단 전용 패널을 일상 운영 가능한 UI로 격상한다. 명령 히스토리(in-memory 50건), 검색/필터, 재실행(Replay), 항목 클릭 시 결과 패널 재표시를 제공한다.

핵심 요구사항:
- in-memory 히스토리 저장소(FIFO 캡) — 순수 로직으로 분리해 단위 테스트 가능
- 실행 시작 시 항목을 `running` 으로 등록, 완료/실패 시 상태 전이 + durationMs 기록
- 검색어 입력에 따른 실시간 필터(부분 일치, 대소문자 무시) — command/args/error 대상
- 항목 클릭 → 결과 패널에 이전 응답 재표시 / 실패 시 에러 박스도 복원
- Replay 버튼 → 동일 명령 재실행
- 기존 WIN-007/WIN-013 패널 기능 회귀 없음

설계 원칙:
- HistoryStore 는 DOM/Electron 의존이 없는 순수 로직 (`desktop/renderer/history-store.ts`) — Node `--test` 로 직접 검증
- 시각/난수/onChange 콜백을 모두 주입 가능하게 하여 테스트 결정성 확보
- `get()` 은 `args` 방어적 복사로 외부 변이로부터 내부 상태 보호

## 3. 실제 변경 내역

### 3.1 신규 파일

#### `desktop/renderer/history-store.ts`
- `HistoryStore` 클래스 + `HistoryEntry` / `HistoryStatus` 타입
- 메서드: `start(command, args?) → id`, `complete(id, response?)`, `fail(id, error, response?)`, `list()`, `get(id)`, `filter(query)`, `capacity()`, `clear()`
- 옵션: `max`(기본 50, 0 이하는 1로 보정), `onChange`, `idFactory`, `now`
- FIFO 정책으로 한도 초과 시 가장 오래된 항목 제거(`unshift` + 길이 `trim`)
- `onChange` 예외는 격리(시스템 안정성)

#### `desktop/__tests__/history-store.test.ts`
- Node `--test` 기반, 서브테스트 12개:
  - start() 정상 등록 / 빈 command 거부
  - complete()/fail() 상태 전이 + durationMs 계산
  - 미존재 id 에 대한 complete()/fail() noop
  - 최신순 정렬 / FIFO 한도 트리밍 / max<=0 보정
  - filter() 부분 일치(대소문자 무시) — command/args/error 대상, 빈 쿼리는 전체
  - get() args 방어 복사
  - clear() 동작 + 빈 상태에서 onChange 미발화
  - onChange 발화 횟수(start/complete/start/fail/clear = 5) + 예외 격리

- `.history-status` 배지(running=노랑, ok=초록, failed=빨강)
- `.history-cmd` monospace + ellipsis
- `.history-meta` 시간/소요시간 표시
- `.history-replay` 보조 버튼 스타일
- `.history-empty.hidden` 상태 처리
- 활성 항목 `.is-active` 강조

#### `desktop/renderer/app.ts`
- `HistoryStore` import, 인스턴스 생성(`max: 50`)
- `activeHistoryId`, `historyQuery` 상태 추가
- `formatHistoryMeta(entry)` — 시간 + 상태(실행 중) 또는 durationMs
- `renderHistory()` — 검색 적용/카운트 갱신/빈 상태 토글/DOM 재구성
- `showHistoryEntry(id)` — 결과 패널 + 에러 박스 복원
- `runCommand` 흐름에 `history.start/complete/fail` + 매 단계 `renderHistory()` 호출
- 이벤트 바인딩:
  - 검색 input → `historyQuery` 갱신 + 재렌더
  - 비우기 버튼 → `history.clear()` + 활성 해제
  - 목록 클릭 위임: Replay 버튼은 `runCommand` 재호출, 항목 클릭은 `showHistoryEntry`
- 초기 1회 `renderHistory()` 호출로 빈 상태 안내 표시
