## 4. 실제 변경 내역

### 4.1 1단계 게이트 문서 신규 작성
`change-winapp-phase1-gate.md`를 신규 생성했다.

문서 구성:
1) 문서 목적 및 판정 원칙
- 증거 기반 PASS/FAIL 기준 명시
- 보류(조건부) 처리 규칙 명시

2) 게이트 메타 정보
- 버전, 범위, 기준 문서, 판정일, 리뷰어, 최종 판정 필드 정의

3) 기능 게이트
- 코어 분리, 이벤트 스트림, Desktop 셸, IPC 왕복, 최소 UI 항목을 표로 정의

4) 보안 게이트
- `contextIsolation`, `nodeIntegration`, preload 최소 API, allowlist, 입력 검증 항목을 표로 정의

5) 빌드/테스트 게이트
- `desktop:build`, `desktop:dev`, IPC 계약 테스트, CLI 빌드, 회귀 엔트리 항목 정의
- 현재 상태 기준 FAIL 항목과 상세 원인 기록

6) 데모 시나리오 게이트
- 앱 기동/정상 명령/실패 명령/preload 상태 시나리오를 표로 정의

7) 종합 판정
- 게이트별 PASS/FAIL 집계
- 최종 판정과 FAIL 해제 조건 명시

8) 승인 체크리스트
- 리뷰어 서명/일시/결과 입력란 포함
