# 작업내역-W002

## 1. 작업 식별 정보
- 작업 ID: WIN-002
- 작업명: CLI 라우터를 코어 실행기로 분리
- 기준 문서: change-winapp-phase1-tickets.md
- 작업 일자: 2026-05-22
- 작업 브랜치: webapp

## 2. 작업 목표
WIN-002의 요구사항에 따라 src/cli/index.ts 내부의 명령 분기 switch 로직을 코어 실행 함수로 이전하고,
main(args)는 파싱과 어댑터 역할 중심으로 축소한다.

핵심 요구사항:
- executeCommand() 신규 작성
- src/cli/index.ts의 switch 분기 로직 코어 함수로 이동
- 기존 동작(에러 출력, 도움말, 종료 처리) 유지

## 3. 실제 변경 내역

### 3.1 신규 파일 추가
- src/core/execute-command.ts

구현 내용:
1) 입력 계약 정의
- ExecuteCommandInput
  - command
  - firstArg
  - rawArgs
  - knownCommands
  - helpText

2) 핸들러 계약 정의
- ExecuteCommandHandlers
- launch, setup, exec, team, state, help 등 기존 switch case와 1:1 대응 핸들러 정의

3) 코어 분기 함수 정의
- executeCommand(input, handlers)
- 기존 main의 switch-case를 코어 함수로 이전
- 기존 default 처리 유지
  - 알 수 없는 옵션 시작 입력(-*)은 launch 경로로 fallback
  - 알 수 없는 명령은 에러/도움말 출력 후 종료
- 기존 예외 처리 유지
  - Error 메시지 출력 후 종료

### 3.2 기존 파일 수정
- src/cli/index.ts

변경 내용:
1) 코어 실행기 import 추가
- ../core/execute-command.js

2) main(args) 내부 역할 정리
- commandArgs = args.slice(1) 추출
- 기존 switch 블록 제거
- executeCommand(...) 호출로 분기 위임

3) 핸들러 클로저 매핑
- 기존 command별 실행 코드를 handlers 객체로 이관
- setup/uninstall 옵션 계산은 기존 로직 유지
- doctor 동적 import 유지
- exec inject 분기 로직 유지
- help 출력 및 unknown flag fallback 동작 유지

4) 타입 정합성 보정
- update 핸들러를 async 래핑하여 Promise<void> 계약 충족

## 4. 동작 보존 관점 확인

1) 옵션 기반 파싱
- flags, options 계산 로직 유지
- commandOwnsLocalHelp 선처리 유지

2) madmax 격리 활성화

- 기존 명령별 함수 호출 경로를 동일 함수로 매핑

4) 오류 처리
- unknown command 처리 및 catch 처리 메시지/종료 동작 유지
