## 2. 목표

- WorkerTransport 전환(WIN-011/012) 이후 잔존하던 tmux 전제 테스트를 OS 프로파일 단위로 분리한다.
- Windows 환경에서는 `tmux` 없이도 회귀 셋이 동일하게 통과해야 한다.
- Linux 환경에서는 기존 `TmuxTransport` 및 호환 검증 테스트가 회귀 없이 유지되어야 한다.

## 3. 변경 내역

### 신규 파일

- [desktop/__tests__/win-smoke.test.ts](../../../desktop/__tests__/win-smoke.test.ts)
  - `LocalProcessTransport` 기반 4개 서브테스트:
    1. `isAvailable()` 가 모든 플랫폼에서 `true`.
    2. Node 자식 프로세스를 spawn → stdout 캡처 → 정상 종료 (`process.execPath` 사용으로 OS 독립).
    3. 허용 목록 밖 명령(`cmd.exe` / `/bin/sh`) 차단 — 보안 경계 회귀 방지.
    4. `send()` 로 stdin 라운드트립 (자식이 `process.stdin.once('data')` 로 echo).

### 변경 파일

- [package.json](../../../package.json)
  - 신규 스크립트 5종:

    - `test:phase2:windows:compiled` — 공통 + `win-smoke.test.js`.
    - `test:phase2:windows` — `build` + `build:desktop` + 위 묶음.
    - `test:phase2:linux:compiled` — 공통 + `sidecar/tmux` + `compat/{doctor-contract,rust-runtime-compat}`.

## 4. 분리 기준

- **공통(common)**: Node `child_process` 또는 순수 TypeScript 모킹만 사용하는 테스트.
  - 데스크탑: `event-bus`, `history-store`, `question-broker`, `hud-view-model`, `sidecar-view-model`, `ipc-contract`.
  - core: `transport`, `local-process-transport`.
  - sidecar `tmux.test.ts` 는 모킹 기반이라 동작 자체는 OS 무관하지만, “tmux 회귀” 의도를 유지하기 위해 Linux 프로파일에만 등록.
- **Windows 전용**: `win-smoke.test.ts` — `LocalProcessTransport` 실제 spawn 동작 + 보안 경계 확인.
- **Linux 전용**: `sidecar/tmux.test.ts`, `compat/doctor-contract.test.ts`, `compat/rust-runtime-compat.test.ts` — 후자 두 개는 fake tmux 바이너리(`#!/bin/sh` + `chmod 0o755`) 를 사용하므로 POSIX 전제.
