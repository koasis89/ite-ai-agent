- 기준 문서: change-winapp-phase2-tickets.md (WIN-013 섹션)

- 작업 브랜치: webapp

## 2. 작업 목표
Main → Renderer 방향의 실시간 이벤트 스트림을 IPC push 채널로 구축하여, 명령 실행 진행 상태(`command.started` / `command.progress` / `command.completed` / `command.failed`)와 패널 상태(`panel.ready`)를 Renderer가 구독해 타임라인에 실시간 반영하도록 한다.

핵심 요구사항:
- IPC 이벤트 push 채널(`omx:event`) 정의 + 구독/해제 채널(`omx:event:subscribe` / `omx:event:unsubscribe`)
- 다중 구독자 관리 EventBus (sender id 키, 데드 sender 자동 정리, send 에러 격리)
- preload `window.omx.subscribeEvents(handler)` / `unsubscribeEvents()` API
- Renderer 타임라인 패널이 push 이벤트로 실시간 갱신
- 단위 테스트로 구독/해제/예외 격리/builder 형상 보장

설계 원칙:
- EventBus는 Electron 의존이 없는 순수 로직 → 노드 `--test`로 직접 검증 가능 (`EventSender` 추상)
- sender 식별은 `WebContents.id` 사용, 동일 id 재구독은 멱등(덮어쓰기)
- 핸들러/`send` 예외는 격리하고 sender를 자동 제거 → 단일 결함이 전파되지 않음
- preload 단에서 `IpcRendererEvent`를 외부에 노출하지 않음(보안) — payload만 전달

## 3. 실제 변경 내역

### 3.1 신규 파일

#### `desktop/ipc/event-bus.ts`
- 순수 로직 EventBus 클래스 + `EventSender` 추상 + 싱글톤 `eventBus`
- 주요 메서드: `subscribe`, `unsubscribe`, `subscriberCount`, `has`, `publish`, `dispose`
- 정적 빌더 `EventBus.event<TDetail>(type, detail?, commandId?)` — `Date.now()` 타임스탬프 자동 부여
- 데드 sender(`isDestroyed?() === true`) 자동 정리
- `send` 예외 발생 시 해당 sender 즉시 제거 + `onSendError` 콜백 전파

#### `desktop/__tests__/event-bus.test.ts`
- Node `--test` 기반, 12 서브테스트:
  - subscribe/publish/unsubscribe 라이프사이클
  - 동일 id 재구독 멱등성(덮어쓰기)
  - 구독자 0명일 때 publish noop
  - destroyed sender 자동 정리
  - send 에러 격리 + `onSendError` 호출 + 해당 sender 제거
  - 잘못된 sender 입력 throw
  - 미등록 id unsubscribe noop
  - `dispose()` 전체 정리
  - `EventBus.event()` 빌더 형상(timestamp 포함, optional 필드 생략)
  - 커스텀 채널명 반영
  - `handleRunCommand` 통합: state_get_status → started→completed 발행
  - `handleRunCommand` 통합: unknown command → failed 발행

### 3.2 변경 파일

#### `desktop/ipc/events.ts`
- IPC 채널 상수에 `EVENT_PUSH = "omx:event"`, `EVENT_SUBSCRIBE = "omx:event:subscribe"`, `EVENT_UNSUBSCRIBE = "omx:event:unsubscribe"` 추가
- `OmxEventType` 유니온(`command.started|progress|completed|failed`, `panel.ready`) + `OmxEvent<TDetail>` 인터페이스 추가
- detail 타입: `CommandStartedDetail`, `CommandProgressDetail`, `CommandCompletedDetail`, `CommandFailedDetail`

#### `desktop/ipc/commands.ts`
- `handleRunCommand`가 명령 처리 흐름에 EventBus 발행 통합:
  - `commandId = cmd-{ts}-{rand}` 생성, `startedAt` 캡처
  - 정상 분기: `command.started` → 명령 실행 → `command.completed`(exitCode/durationMs 포함)
  - allowlist 거부: `command.failed`(메시지 포함)
  - 예외: `command.failed`(error.message 포함)

#### `desktop/main/index.ts`
- `eventBus` import 추가
- `registerIpcHandlers`:
  - `ipcMain.on(EVENT_SUBSCRIBE)` → `eventBus.subscribe({id, send, isDestroyed})` + `WebContents.once("destroyed")`로 자동 unsubscribe 훅
  - `ipcMain.on(EVENT_UNSUBSCRIBE)` → `eventBus.unsubscribe(sender.id)`
  - 핫 리로드 안전성 위해 `removeAllListeners` 선행

#### `desktop/preload/index.ts`
- `subscribeEvents(handler) → unsubscribe` API 추가
  - 핸들러 다중 등록 지원(참조 카운팅): 첫 구독 시 Main에 `EVENT_SUBSCRIBE`, 마지막 해제 시 `EVENT_UNSUBSCRIBE` 송신
  - `ipcRenderer.on(EVENT_PUSH)` 단일 리스너로 fanout, 외부에는 `OmxEvent` payload만 전달(IpcRendererEvent 노출 안 함)
  - 핸들러 throw 격리
- `unsubscribeEvents()` 일괄 해제 API 추가
- `contextBridge.exposeInMainWorld("omx", { runCommand, subscribeEvents, unsubscribeEvents })`

#### `desktop/renderer/app.ts`
- `Window.omx` 타입 선언에 `subscribeEvents` / `unsubscribeEvents` 추가
- `formatEventLine(event)` 포맷터로 이벤트 타입별 타임라인 문구 생성
- 시작 시 `window.omx.subscribeEvents(...)` 호출 → `addTimeline`으로 라우팅
- `beforeunload`에서 unsubscribe 정리

### 3.3 빌드 인프라 보정 (W012 잠재 회귀 정정)

  - `rootDir` 제거(자동 추론) + `include`에 `src/core/transport.ts`, `src/core/local-process-transport.ts` 명시
  - 결과 outDir 구조: `dist-desktop/desktop/...` + `dist-desktop/src/core/...`
- `desktop/scripts/copy-assets.mjs`
  - 렌더러 자산 복사 대상을 `dist-desktop/desktop/renderer/`로 갱신
- `package.json`
  - `desktop:dev` 진입점: `dist-desktop/desktop/main/index.js`
  - `test:phase1:ipc-contract` 경로: `dist-desktop/desktop/__tests__/ipc-contract.test.js`
