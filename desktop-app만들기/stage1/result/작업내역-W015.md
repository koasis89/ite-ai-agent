- 티켓: WIN-015 / User Story #112 (US-15) / Sprint SP-02 / Epic EP-01

## 2. 목표
CLI의 readline 기반 블로킹 질문 루프를 Electron Renderer 모달로 대체할 수 있는 IPC 인프라를 구축한다. 단, 기존 CLI 흐름(`runQuestionUi`, tmux pane 전략)은 회귀 없이 격리 유지한다.

## 3. 변경 내역

## 6. 후속
- **WIN-016+**: Desktop Main이 실제 `omx question` 자식 프로세스를 spawn할 때 `OMX_QUESTION_RENDERER=desktop-ipc`를 주입하고 자식 프로세스 stdio/파일 워치를 통해 `questionBroker`와 연결하는 cross-process 브리지가 필요. WIN-015는 *브리지 한쪽 끝(브로커 + 모달 + 채널/전략 enum)*만 구축.
- Renderer 측에서 다국어 라벨/단축키 안내가 필요할 수 있음 — UX 폴리시 정의 후 추가.
