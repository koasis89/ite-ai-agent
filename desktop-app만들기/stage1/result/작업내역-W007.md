## 4. 실제 변경 내역

### 4.1 명령 입력/실패 시나리오를 위한 Gateway 보강
`desktop/ipc/commands.ts`
- 요청 스키마의 `command`를 문자열로 수용하도록 조정
- allowlist 검사(`state_get_status`만 허용)를 런타임 검증으로 분리
- 허용 외 명령은 `UNKNOWN_COMMAND` 에러 응답 반환

의도:
- UI에서 입력 명령을 자유롭게 받아 실패 케이스까지 시각적으로 검증 가능
- 실제 실행 허용은 기존 정책대로 단일 명령으로 제한

### 4.2 WIN-007 진단 패널 UI 구조 추가
`desktop/renderer/index.html`
- 기존 단순 상태 화면을 2패널 레이아웃으로 확장
  - 좌측: 명령 실행 패널
  - 우측: preload 상태 + 명령 결과 패널
- UI 요소 추가
  - 명령 입력 필드 (`#command-input`)
  - 실행 버튼 (`#run-command`)
  - 실패 시나리오 버튼 (`#run-invalid`)
  - 에러 박스 (`#error-box`)
  - 이벤트 타임라인 (`#event-timeline`)
  - 결과 출력 (`#command-result`)

### 4.3 스타일 분리
`desktop/renderer/styles.css` 신규 생성
- WIN-007 레이아웃/컴포넌트 스타일 추가
- 카드형 패널, 버튼 상태, 에러 박스, 타임라인 스타일 정의
- 모바일 대응(`max-width: 900px`) 반응형 레이아웃 포함

### 4.4 렌더러 로직 구현
`desktop/renderer/app.ts`
- 공통 `runCommand(command)` 실행 흐름 추가
- 성공/실패 모두 결과 JSON 렌더
- 실패 시 에러 박스 표시
- 이벤트 타임라인 기록
  - `command.started`
  - `command.completed`
  - `command.failed`
  - 초기 `panel.ready`
- 실패 시나리오 버튼 클릭 시 의도적으로 비허용 명령 호출

### 4.5 빌드 자산 복사 보강
`desktop/scripts/copy-assets.mjs`
- 기존 HTML 복사에 `styles.css` 복사 추가
- `build:desktop` 결과물에서 스타일 누락되지 않도록 보완

- `node -e "import('./dist-desktop/ipc/commands.js').then(async (m) => { const ok = await m.handleRunCommand({ command: 'state_get_status' }); const bad = await m.handleRunCommand({ command: 'state_get_status_invalid' }); console.log(JSON.stringify({ ok, bad }, null, 2)); }).catch((e) => { console.error(e); process.exit(1); });"`

결과:

- 비허용 명령: `ok: false` + `UNKNOWN_COMMAND` 에러 반환
