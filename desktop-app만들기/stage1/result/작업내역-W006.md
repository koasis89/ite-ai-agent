# 작업내역-W006

## 1. 작업 식별 정보
- 작업 ID: WIN-006
- 작업명: IPC Command Gateway (read-only 최소 기능)
- 기준 문서: change-winapp-phase1-tickets.md (WIN-006)
- 작업 일자: 2026-05-22
- 작업 브랜치: webapp

## 2. 작업 목표
WIN-006 목표에 맞춰 Renderer가 안전한 경로를 통해 단일 read-only 명령(`state_get_status`)을 호출하도록
IPC Gateway를 구성하고 결과 JSON을 UI에 렌더한다.

핵심 요구사항:
- IPC 채널 정의 및 핸들러 등록
- preload에서 `window.omx.runCommand(request)` API 노출
- allowlist 적용 (`state_get_status`만 허용)
- 입력값 기본 검증(zod)
- Renderer 버튼 클릭 시 결과 JSON 표시

## 4. 실제 변경 내역

### 4.1 IPC 채널/인터페이스 정의
1) `desktop/ipc/events.ts`
- IPC 채널 상수 정의
  - `omx:run-command`
- 공용 envelope 타입 정의

2) `desktop/ipc/commands.ts`
- `zod` 기반 요청 검증 스키마 추가
- 허용 명령 allowlist를 `state_get_status` 단일 값으로 고정
- `handleRunCommand(rawRequest)` 구현
  - 유효성 실패 시 `INVALID_REQUEST` 반환
  - 허용 명령 처리 시 상태 JSON 반환
  - 처리 예외 시 `COMMAND_FAILED` 반환

### 4.2 Main 프로세스 IPC 핸들러 등록
`desktop/main/index.ts`
- `ipcMain` 사용으로 `omx:run-command` 핸들러 등록
- 앱 재기동/핫리로드 상황을 고려해 `removeHandler` 후 `handle` 재등록
- 핸들러는 `handleRunCommand`를 호출해 검증/allowlist 로직을 단일화

### 4.3 Preload 브리지 API 노출
`desktop/preload/index.ts`
- `ipcRenderer.invoke` 기반 `runCommand` 함수 추가
- `window.omx.runCommand(request)` API 노출
- 기존 `window.omxDesktop` 상태 노출은 유지

### 4.4 Renderer 호출/결과 렌더 연결
1) `desktop/renderer/index.html`
- `state_get_status 실행` 버튼 추가 (`#run-status`)
- 명령 결과 출력 영역 추가 (`#command-result`)

2) `desktop/renderer/app.ts`
- `window.omx.runCommand({ command: "state_get_status" })` 호출 추가
- 버튼 클릭 시 응답 JSON을 화면에 렌더
- 호출 예외 시 에러 JSON 렌더
