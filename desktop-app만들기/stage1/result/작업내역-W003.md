## 2. 작업 목표

CLI adapter(`src/cli/index.ts`)에서만 출력/종료코드 처리를 담당하도록 경계를 명확히 한다.

## 4. 실제 변경 내역

### 4.1 src/core/execute-command.ts
핵심 변경:
1) 코어 반환 계약 추가
- `ExecuteCommandOutcome` 인터페이스 신규 정의
  - `ok`
  - `exitCode`
  - `errorMessage?`
  - `unknownCommand?`
  - `showHelp?`

2) 함수 반환형 변경
- 기존: `Promise<void>`
- 변경: `Promise<ExecuteCommandOutcome>`

3) 명령 처리 결과 반환으로 전환
- 각 case 처리 후 `return { ok: true, exitCode: 0 }` 반환
- unknown command 시 코어에서 출력/종료하지 않고 아래 값 반환
  - `ok: false`
  - `exitCode: 1`
  - `unknownCommand`
  - `showHelp: true`

4) 예외 처리 경계 정리
- 기존 코어의 `console.error(...)`, `process.exit(1)` 제거
- 예외는 `errorMessage` 포함 outcome으로 반환

적용 효과:
- 코어는 판단/결과만 반환
- 출력과 종료코드는 어댑터 계층에서 일원화 가능

### 4.2 src/cli/index.ts
핵심 변경:
1) `executeCommand(...)` 결과 수신
- `const outcome = await executeCommand(...)` 형태로 변경

2) CLI adapter에서 출력/종료 처리
- `outcome.ok === false`일 때만 처리:
  - `errorMessage` 있으면 `console.error`
  - `unknownCommand` 있으면 `console.error`
  - `showHelp`면 `console.log(HELP)`
  - `process.exitCode = outcome.exitCode`

적용 효과:
- CLI adapter가 출력/종료 책임을 단일 지점에서 관리
- 코어 로직과 CLI IO 책임 분리 강화
