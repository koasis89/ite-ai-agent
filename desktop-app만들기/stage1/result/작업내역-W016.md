## 2. 목표
tmux 의존 HUD 렌더(`src/hud`)를 데스크탑 Renderer 의 DOM 패널로 전환한다. Model(상태 파일/이벤트) → ViewModel(순수 로직, Electron 의존 0) → View(DOM 패널) 의 MVVM 패턴을 도입하여 CLI HUD 와 격리된 데스크탑 전용 경로를 구축한다.

## 3. 변경 내역

### 신규
- `desktop/ipc/hud.ts` — `HudSnapshot` 타입 / `normalizeHudContext()` / `HudViewModel` 클래스 / `stripAnsi()` 헬퍼. Electron·DOM 의존 0, 주입 가능한 `now()`.
- `desktop/renderer/hud-panel.ts` — `HudViewModel` 을 구독해 DOM 노드만 갱신하는 어댑터(`HudPanel` 클래스).

### 변경
- `src/hud/index.ts` — ViewModel 친화 API 재노출(`readAllState`, `readHudConfig`, `DEFAULT_HUD_CONFIG`, `HudRenderContext` 등) + `getHudSnapshot(cwd, config?)` 추가. CLI 렌더 경로(`renderHud`, `runWatchMode`, `hudCommand`) 무변경.
- `desktop/ipc/events.ts` — `IPC_CHANNELS` 에 `HUD_SNAPSHOT`/`HUD_WATCH`/`HUD_UNWATCH` 채널 추가, `OmxEventType` 에 `"hud.update"` 추가.
- `desktop/main/index.ts` — `getHudSnapshot` import + `registerHudHandlers()` 추가. `HUD_SNAPSHOT`(invoke), `HUD_WATCH`(send → 1s interval 폴링 → `eventBus.publish({type:"hud.update", ...})`), `HUD_UNWATCH` 처리. `webContents.destroyed` 시 watcher 자동 해제.
- `desktop/preload/index.ts` — `window.omx.getHudSnapshot()` / `window.omx.watchHud()` 노출.
- `desktop/renderer/app.ts` — `HudViewModel`/`HudPanel` 인스턴스화, 초기 `getHudSnapshot()` 적용, `watchHud()` 호출, push 이벤트 핸들러에서 `hudViewModel.applyEvent(event)` 호출. `beforeunload` 에서 자원 해제.
- `desktop/renderer/index.html` — `<section id="panel-hud">` 마크업 추가(phase/session/git/tokens/queue/workers/last event/notify slot).
- `desktop/renderer/styles.css` — `.panel-hud`, `.hud-head`, `.hud-grid`, `.hud-notify`, `data-status` 상태별 배지 색상 규칙 추가.
