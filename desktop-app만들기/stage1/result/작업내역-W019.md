## 4. 코드서명 정책 (placeholder)

- 현 단계: 인증서 미발급 → `CSC_IDENTITY_AUTO_DISCOVERY=false` 로 빌드.
- 도입 시: `electron-builder.yml` 의 주석 처리된 `win.certificateFile` / `win.certificatePassword` 를 활성화하거나, CI 환경에서 `CSC_LINK` (base64-encoded .pfx) + `CSC_KEY_PASSWORD` 환경변수만 설정해도 electron-builder 가 자동으로 서명한다.
- 검증 흐름: 서명 후 `signtool verify /pa /v` 또는 PowerShell `Get-AuthenticodeSignature` 로 확인.

## 5. 검증

### 설정 정합성 테스트 (이번 머신에서 즉시 실행 가능)

```
npm run test:phase2:windows:compiled
# common  → tests 18 / pass 18 / fail 0  (packaging-config 포함)
# smoke   → tests 1  / pass 1  / fail 0
```

### 실제 NSIS 빌드

- 본 개발 머신(비-관리자 PowerShell, Windows Developer Mode 비활성) 에서는 electron-builder 가 `winCodeSign` 캐시 내부의 darwin .dylib **심볼릭 링크** 를 추출하지 못해 빌드 실패 (Windows 의 심볼릭 링크 생성 권한 제약).
  - 에러 시그니처: `7za x ... errorOut=ERROR: Cannot create symbolic link ... darwin/10.12/lib/libcrypto.dylib`.
  - 해소 방법(작업자 환경 의존):
    1. Windows **Developer Mode** 활성화 (Settings → Privacy & security → For developers), 또는
    2. **관리자 권한** PowerShell 에서 `npm run desktop:package` 실행, 또는
    3. CI 러너에서 위 두 조건 중 하나를 만족한 상태로 실행.
- 위 조건이 충족된 환경에서는 `release/oh-my-codex-<version>-x64-setup.exe` 가 생성되며, 본 티켓의 모든 설정/스크립트는 그 상태에서 그대로 사용 가능하다.
