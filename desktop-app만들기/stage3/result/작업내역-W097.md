# 작업내역 ST3-WIN-097: 스트리밍/타임아웃/재시도 정책 구현

## 1. 개요
LLM Gateway 호출의 안정성과 사용자 경험을 개선하기 위해 스트리밍 이벤트 계약을 확정하고, 타임아웃 및 재시도(Retry) 메커니즘을 IPC 계층에 구현하였습니다.

## 2. 주요 변경 사항

### 2.1 이벤트 계약 (IPC/Events)
- `llm.stream.chunk`: 토큰 단위 텍스트 전달
- `llm.stream.tool_call`: 도구 호출 정보 전달
- `llm.stream.done`: 생성 완료 및 최종 통계 전달
- `llm.stream.error`: 에러 발생 및 재시도 가능 여부 전달
- `llm.stream.retry`: 재시도 시도 정보 전달

### 2.2 IPC 핸들러 (LLM Gateway)
- `AbortController`를 이용한 타임아웃 제어 (기본 30초)
- 지수 백오프(Exponential Backoff)를 적용한 재시도 루프 구현 (최대 3회/전역 설정 가능)
- 스트리밍 모드(`stream: true`) 지원 및 실시간 이벤트 발행 로직 추가
- TTFT(Time To First Token) 메트릭 측정 로직 포함

### 2.3 메인 프로세스 부트스트랩
- `LlmGatewayBackend` 인터페이스 확장 (`invoke` 시 `abortSignal` 수신, `stream` 메서드 추가)
- Mock 어댑터(`mock-echo`, `mock-reverse`)에 스트리밍 시뮬레이션 로직 보강

## 3. 검증 결과
- **타임아웃**: 강제 지연 시 `TIMEOUT` 에러 코드 반환 및 재시도 동작 확인
- **스트리밍**: Mock 어댑터를 통한 토큰 단위 UI 업데이트 이벤트 발행 확인
- **재시도**: 실패 시나리오에서 `llm.stream.retry` 이벤트와 함께 자동 재집행 확인
