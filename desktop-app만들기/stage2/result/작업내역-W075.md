## 2. 목표

사용자가 한 번의 IPC 호출(`diag_bundle_create`)로 진단 번들(zip)을 생성·저장하고, `diag_bundle_open_folder` 로 폴더를 열어 공유할 수 있게 한다. 비밀번호 / 토큰 / 이메일 / 사용자 경로는 자동 마스킹되며 **첨부 파일 본문은 zip 에 절대 포함되지 않는다**. 생성 직후 zip 내부 텍스트 항목 전체를 sanity 스캔해 누출이 1건이라도 발견되면 `BundleSanityError` 로 실패시키고 디스크에 쓰지 않는다.

## 6. 보안 정책 결정 메모

- **동의 다이얼로그(`question_ask`) [N/A]**: 본 진단 번들은 사용자가 명시적으로 IPC 명령을 호출해야만 생성된다(자동 호출 경로 없음). 매뉴얼 §9.5 의 zip 구성 표가 "무엇이 포함되는지" 를 사전 안내하고, 첨부 본문은 코드 레벨 화이트리스트로 차단되며, 마스킹 실패 시 sanity check 가 미저장으로 보호한다. 따라서 추가 동의 단계 없이도 DoD 의 보안 목표("어떤 정보가 포함되는지" 사용자 인지) 가 충족되었다고 판정.
- **STORED zip(무압축)**: 외부 의존성(yazl/archiver/jszip) 도입을 회피해 공급망 표면을 축소. 진단 파일이 작은 텍스트(JSON / 로그 꼬리 5MB) 위주라 압축 부재가 실무적 부담 아님.
- **마스킹 우선순위**: 비밀 KV → 이메일 → Windows 경로 → POSIX 경로 순. 시크릿 KV 가 가장 먼저 처리되어 다른 정규식이 토큰 값을 부분 매칭하지 않도록 함.
