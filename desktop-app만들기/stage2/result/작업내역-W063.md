## 1. 목표 요약

첨부 칩(WIN-061/062) 클릭/Enter 시 첨부 mime 에 맞춰 인라인 프리뷰를 우측 컨텍스트 패널에 표시한다. 텍스트는 main 프로세스의 `attachment_read_text` IPC 로 최대 64KB 까지만 읽으며, image/PDF 는 sandbox 화된 `file://` 또는 blob URL 로 표시한다. 모든 file:// 로드는 main 의 화이트리스트(attachments dir + renderer asset dir) 안으로 제한된다.


### 3.2 단위 테스트

```
npm run test:phase2:common:compiled
```

전후 비교 (게이트 누적 카운트):
- W062 게이트 직후: 212 tests / 159 pass / 0 fail / 53 skipped.
- **W063 게이트 직후: 236 tests / 180 pass / 0 fail / 56 skipped.**
- 증분: +24 tests (+21 pass / +3 skip).

신규 `preview.test.ts` 케이스 (20+):
- `clampMaxBytes`: NaN/non-number → default, 0/음수 → MIN, 분수 floor, 1M → cap.
- `detectEncoding`: UTF-16LE BOM / default UTF-8.
- `readBufferAsText`: UTF-8 plain / UTF-8 BOM 제거 / UTF-16LE BOM 디코드 / 2KB→64B truncated.
- `sanitizeSvg`: `<script>` 제거 / `on*` 제거 / `<foreignObject>+<iframe>` 제거 / `javascript:` href 제거 / 외부 https href 제거 / benign svg 미변경.
- `PreviewLruCache`: capacity guard / overflow evict / get-MRU 승격.
- `isTextMime`: text/plain / application/json / image/png false / 확장자 fallback.
- `AttachmentRepo.readText`: 64KB 이내 정상 / 100KB → truncated:true & totalBytes=100000 / bad id → null (driver 가용 시; 부재 환경은 t.skip).

확장된 `ipc-contract.test.ts` (3 subtests):
- accepts `attachment_read_text` with valid id (default cap) → ok=true, text="hello", truncated=false.
- rejects `attachment_read_text` with bad id pattern → INVALID_REQUEST.
- rejects `attachment_read_text` with `maxBytes` out of range (1_000_000) → INVALID_REQUEST.

### 3.3 보안 회귀

- 텍스트 읽기 64KB 상한: `readBufferAsText` 의 `cap` 분기 + repo 의 `clampMaxBytes` 이중 적용 → 100KB 입력 시 `bytesRead ≤ 65_536`, `truncated:true`, `totalBytes=100000` 확인.
- 경로 탈출 차단: `AttachmentRepo.readText("../etc/passwd", {})` → `isValidAttachmentId` 거부로 null.
- IPC 스키마: `ATTACHMENT_ID_RE` 위반 (`../../escape`) → INVALID_REQUEST.
- maxBytes 범위 외 (>65536) → INVALID_REQUEST.
- SVG 활성 컨텐트: 5종 회귀 패턴(script/on*/foreignObject/javascript:href/external href) 모두 제거 확인.
- file:// allowlist: `installFilePrefixGuard` 가 attachments/renderer 외 path 의 file:// 요청을 cancel. Windows `/C:/…` 슬래시 정규화로 false-positive 회피.
- CSP 메타: `script-src 'self'` 로 inline 스크립트 차단(기존 IIFE 는 모두 module 로드 → 안전), `connect-src 'self'` 로 외부 네트워크 차단.
