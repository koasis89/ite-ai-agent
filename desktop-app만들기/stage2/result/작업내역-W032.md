## 2. 목표 (티켓 인용)

- 좌측 패널에서 **세션(대화 그룹) CRUD + 활성 세션 전환**.

## 4. 핵심 API 요약

```ts
// session-store.ts
export interface Session {
  readonly id: string;
  name: string;
  readonly createdAt: number;
  updatedAt: number;
  messageIds: string[]; // 스펙 호환 — Phase 5 영속화에서 활용
}

export const SESSION_NAME_PATTERN = /^[\p{L}\p{N} _\-()]{1,64}$/u;

export class SessionStore {
  create(name: string): Session;            // 이름 무효 시 SessionNameError
  rename(id: string, name: string): Session;
  delete(id: string): boolean;              // 활성이면 active=null, 스냅샷도 정리
  select(id: string | null): Session | null;
  list(): readonly Session[];
  active(): Session | null;
  saveSnapshot(id: string, messages: readonly ChatMessage[]): void;
  loadSnapshot(id: string): ChatMessage[];
  onChange(handler): () => void; // 사유: create/rename/delete/select/snapshot
}

// session-list.ts — DOM
export class SessionList {
  constructor(elements: { root; list; newButton; emptyHint? }, options?);
  storeRef(): SessionStore;
  dispose(): void;
}
export function bindSessionToChat(
  store: SessionStore,
  getChatMessages: () => readonly ChatMessage[],
  replaceChat: (messages: readonly ChatMessage[]) => void,
): () => void;
```

### 4.1 활성 세션 전환 흐름

1. 사용자가 좌측에서 다른 세션 클릭 → `SessionStore.select(newId)` 호출.
2. `bindSessionToChat` 구독자가 `select` 이벤트 수신.
3. **이전 세션의 메시지를 `saveSnapshot(lastActiveId, vm.list())` 로 보존**.
4. **새 세션의 스냅샷을 `vm.replace(loadSnapshot(newId))` 로 즉시 교체** → ChatPanel 이 `'replace'` 이벤트로 단일 렌더.
5. 메시지 손실 없이 양방향 전환 가능.

### 4.2 단축키

| 키 | 동작 | 활성 조건 |
|---|---|---|
| `Ctrl+N` | 새 세션 생성 (이름 prompt) | 좌측 패널 내부 활성 또는 신규 버튼 포커스 |
| `F2` | 활성 세션 이름 변경 | 좌측 패널 내부 활성 |
| `Delete` | 활성 세션 삭제 (`QuestionModal` 확인 후) | 좌측 패널 내부 활성 |

→ 채팅 입력창(`<textarea>`)에서의 `Enter`/`Delete` 와 충돌하지 않도록 `document.activeElement` 가 `#session-pane` 내부일 때만 동작.

## 6. 후속 작업

- **매뉴얼 §5 갱신**: 사용자 정책상 매뉴얼(`winapp만들기/stage1/winapp-manual.md`)은 본 티켓에서 편집하지 않음. W030/W031 와 동일 정책 승계, 일괄 갱신 시점에 §5 에 다음 항목 반영 필요:
  - 좌측 패널 사용법 (신규/선택/이름변경/삭제)
  - 단축키 표 (`Ctrl+N` / `F2` / `Delete`)
  - 세션 이름 규칙 (`^[\p{L}\p{N} _\-()]{1,64}$`)
- **WIN-033**: 슬래시 명령 라우터 — 현재 ChatPanel 의 `/...` 입력은 system stub 만 반환. 본 티켓 완료 후 진입 가능.
- **WIN-034**: 응답 스트리밍 source — 데모 `setInterval` echo 를 실제 `command.progress` / PTY `onData` 로 교체.
- **Phase 5 영속화**: 현재 `SessionStore` 는 in-memory. `messageIds` 필드는 SQLite 어댑터 도입 시 재사용. 인터페이스 변경 없이 백엔드만 교체 가능하도록 설계.
