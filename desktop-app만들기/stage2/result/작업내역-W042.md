## 2. 목표
세션 / 메시지 / 도구 호출 로그를 `app.getPath('userData')/data.sqlite` 단일 파일에 영속 저장하고, prepared statement + 1MB content 상한 + FK cascade + WAL 모드를 강제한 main 측 SQLite 백엔드 계층을 신설한다. Phase 4 의 renderer 측 `session-store.ts`(in-memory) 와 동일한 인터페이스(create/rename/delete/list/get + message append/list/updateStreaming) 를 main 측 리포에 구현해 후속 IPC 어댑터 교체의 기반을 마련한다.

## 4. 핵심 API·정책
- **경로 SSOT**: `app.getPath('userData')/data.sqlite` — Windows: `%APPDATA%/oh-my-codex/data.sqlite`.
- **SQL 인젝션 차단**: 모든 쿼리는 prepared statement (`db.prepare(...)`). 테스트(`SQL 인젝션 시도는 prepared statement 로 무력화`)가 회귀 보장.
- **세션 이름 검증**: `SessionNameError` — 제어문자(U+0000~001F, U+007F) 차단 + `^[\p{L}\p{N} _\-()]{1,64}$` (renderer SSOT 와 동일).
- **content 1MB 상한**: `ContentTooLargeError(bytes)` — `MessageRepo.append` / `updateStreaming` / `ToolCallRepo.append(args)` 모두 적용.
- **Cascade 삭제**: `sessions` 삭제 → FK ON DELETE CASCADE 로 `messages` + `tool_calls` 일괄 제거. 테스트 회귀.
- **WAL 모드**: `PRAGMA journal_mode=WAL` + `synchronous=NORMAL` — 디스크 디바이스 / `:memory:` 분기 처리.
- **마이그레이션**: `migrations/NNN_*.sql` 패턴만 인식 + 사전식 정렬 + `schema_migrations` 테이블 멱등 기록 + 잘못된 파일명 건너뜀.
- **driver injection**: `OpenDatabaseOptions.driverFactory` — 단위 테스트가 모의 driver 주입 가능.

## 6. 후속 (분리된 작업)
- **WIN-042b (예정)**: Phase 4 의 `desktop/renderer/chat/session-store.ts` 를 IPC 어댑터로 교체. 현재 renderer 는 동기 API 이므로 (a) preload 에 새 IPC 채널(session.*, message.*) 노출, (b) renderer 가 부팅 시 IPC 로 초기 hydrate 후 변경마다 fire-and-forget persist 패턴 도입이 필요. main 측 영속 백엔드(SessionRepo/MessageRepo/ToolCallRepo) 는 본 티켓에서 완성되었으므로 분리해도 안전.
- 매뉴얼 §6 "데이터 저장 경로" 갱신은 사용자 정책에 따라 수동 작업으로 남김(체크리스트 [ ] 유지).
- 성능 회귀(1000 세션 × 500 메시지 < 200ms) 측정은 native binding 이 정상 적재된 환경(예: Electron 패키지 dev 빌드 또는 x64 CI) 에서 별도 fixture 로 측정 예정.
