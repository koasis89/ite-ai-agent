# 작업내역 — WIN-074: 첫 실행 마법사 (작업 디렉터리 / 기본 설정 / 옵트인 텔레메트리)

## 1. 식별

- 티켓: WIN-074 (Phase 8 / UX)
- 선행: WIN-041 (ConfigStore) / WIN-053 (권한 다이얼로그 패턴) / WIN-071·WIN-072 (업데이트 채널)
- 후속: WIN-075 (진단 번들 — 텔레메트리 옵트인과 정합)
- 작업 일자: 2025-Q4 (Phase 8 4번째 티켓)
- Sprint: SP-08 / Epic: EP-02 (REUSE — W071/W072/W073 동일)

## 2. 목표

신규 설치 후 첫 실행 시 4단계 마법사(Welcome → Workspace → Updates → Telemetry → Done) 로
안전한 기본값을 강제하고, **시스템 디렉터리 / UNC / 드라이브 루트** 같은 위험한 작업
디렉터리를 차단하며, 텔레메트리는 **명시 옵트인 + 화이트리스트(`appVersion` / `os` /
`commandCount`)** 외에는 어떠한 정보도 송신되지 않도록 게이트한다. 중단 후 재실행 시
마지막 완료 단계의 다음 단계부터 이어가도록 상태를 `<userData>/first-run.json` 에 분리
저장한다(ConfigStore schema v1 무결성 유지 — schema v2 마이그레이션 회피).

## 3. 구현

### 3.1 첫 실행 상태 저장소

[desktop/main/first-run/state.ts](../../../desktop/main/first-run/state.ts) (신규)

- `WIZARD_STEPS = ['welcome','workspace','updates','telemetry','done']` 5단계 화이트리스트
  + `isWizardStepId` / `coerceWizardStepId` / `nextStepAfter` 헬퍼.
- `validateWorkspacePath(p, opts)` — 거부 규칙:
  - `^[\\/]{2}` UNC → `UNC network share not allowed`
  - `path.isAbsolute === false` → `workspace path must be absolute`
  - `^[A-Za-z]:[\\/]?$` 드라이브 루트 → `drive root not allowed`
  - `C:\Windows` / `C:\Program Files` / `C:\Program Files (x86)` / `*\ProgramData` (대소문자 무시)
    → `system directory not allowed`
  - `fs.stat`/`fs.access(W_OK)` 실패 → `workspace path does not exist` / `not writable`
  - 테스트 주입: `opts.fs`(stat/access) + `opts.skipExistenceCheck`.
- `validateStepData(stepId, raw, opts)` — 단계별 zod-free 검증:
  - `workspace`: `{ path: string }` → `validateWorkspacePath` 통과 후 정규화 경로 반환.
  - `updates`: `{ enabled: boolean, channel: stable|beta|alpha }` 또는 `{ skipped: true }` (스킵 시 기본 stable).
  - `telemetry`: `{ optIn: boolean, endpoint?: https://… }` 또는 `{ skipped: true }`. http 거부.
  - `welcome`/`done`: 빈 객체.
- `FirstRunStore` 클래스:
  - `load()` — `userData/first-run.json` 읽기. ENOENT → 기본값. 파싱 실패 → 기본값 복원 + 로그.
  - `snapshot()` — `{ completed, currentStep, data }` 얕은 복사.
  - `completeStep(stepId, raw)` — `validateStepData` 통과 후 `data[stepId]` 갱신 +
    `currentStep = nextStepAfter(stepId)` (단, `stepId === 'done'` 이면 `completed=true`).
    영속화는 `tmp + rename` 원자적 쓰기.
  - `reset()` — 초기 상태 복귀(테스트/지원).
  - `isTelemetryAllowed()` — 다음 모두 true 일 때만 true: 로드 완료 + `completed=true` +
    `telemetry.optIn=true` + `skipped !== true`.
  - `sanitizeTelemetryPayload(input)` — 옵트인 차단 시 `null` 반환(네트워크 호출 게이트).
    허용 시 화이트리스트(`appVersion`/`os`/`commandCount`) 외 모든 키 제거.

    위임. 백엔드 미초기화 시 `ok=false, applied=false, reason="wizard backend not initialised"`.
- `perCommandArgValidators` 에 2건 등록.

### 3.4 마법사 렌더러

신규 6개 파일 — DOM-light 직접 조작, `innerHTML` 미사용 (textContent / value 만).

- [desktop/renderer/first-run/Wizard.ts](../../../desktop/renderer/first-run/Wizard.ts) — 컨테이너
  (`role="dialog"`, `aria-modal="true"`, `aria-live="polite"` 상태 영역). `wizard_state_get`
  으로 현재 단계 조회 → 적절한 step 마운트 → 입력 수집 → `wizard_step_complete` 호출 →
  `completed=true` 시 `onComplete()` 호출.
- [Welcome.ts](../../../desktop/renderer/first-run/steps/Welcome.ts) — 앱 소개 + 보안 정책 1줄
  요약(시스템 dir/UNC 차단 + 텔레메트리 화이트리스트 명시) + "시작" 버튼.
- [Workspace.ts](../../../desktop/renderer/first-run/steps/Workspace.ts) — 절대 경로 입력 폼.
  필수(스킵 불가). 비어 있으면 포커스 이동.
- [Updates.ts](../../../desktop/renderer/first-run/steps/Updates.ts) — `enabled` 체크박스
  (기본 on) + 채널 `<select>` (stable/beta). 건너뛰기 버튼 → `{skipped: true}`.
- [Telemetry.ts](../../../desktop/renderer/first-run/steps/Telemetry.ts) — 화이트리스트 표
  (appVersion / os / commandCount 만 송신됨을 명시) + `optIn` 체크박스(기본 off) +
  건너뛰기 버튼(`{skipped:true}` — 송신 안 함).
- [Done.ts](../../../desktop/renderer/first-run/steps/Done.ts) — 선택값 요약 표 + "마침".
