# 작업내역 — WIN-073: Authenticode 코드서명 (EV/OV, self-signed fallback)

## 1. 식별

- 티켓: WIN-073 (Phase 8 / Security)
- 선행: WIN-019 (NSIS) / WIN-071 (자동 업데이트 통합)
- 후속: WIN-074 (첫 실행 마법사)
- 작업 일자: 2025-Q4 (Phase 8 3번째 티켓)
- Sprint: SP-08 / Epic: EP-02 (REUSE — W071/W072 동일)

## 2. 목표

NSIS 설치형 `.exe` 와 자동 업데이트 산출물을 Authenticode SHA-256 으로 서명하고,
electron-updater 의 `verifyUpdateCodeSignature: true` 와 결합해 publisherName 매칭 실패 시
업데이트 적용을 거부한다. 인증서/패스프레이즈는 환경 변수 + CI Secrets 로만 주입하고,
저장소 평문 노출을 lint 로 강제 차단한다. self-signed 경로는 PoC 한정으로 명시적 opt-in
한다.

## 3. 구현

### 3.1 코드서명 런타임 설정 파서

[desktop/main/updater/codesign.ts](../../../desktop/main/updater/codesign.ts) (신규)

- `parseCodesignConfig({env})` — env 변수 → `CodesignRuntimeConfig` 변환 + 경고 수집.
  - `WIN_CSC_LINK` 검증: `https://` 또는 안전한 로컬 경로 정규식 (`^([A-Za-z]:[\\/]|[.][\\/]|[\\/])`)
    만 허용. `http://` / `file://` 거부.
  - `WIN_CSC_KEY_PASSWORD` 필수 — 누락 시 `enabled: false` 로 강등 + 경고.
  - `WIN_CSC_TIMESTAMP_URL` — http(s) 만 허용. 그 외는 기본 `http://timestamp.digicert.com` 폴백.
  - `WIN_CSC_VERIFY_UPDATE_SIGNATURE` — 기본 `true`. 명시 `0/false/no` 만 OFF.
  - `WIN_CSC_ALLOW_SELF_SIGNED` — `isTruthy` 헬퍼로 `1/true/yes/on` 만 허용.
- `describeCodesignConfig(config)` — 안전 로깅 직렬화 (`pass` / `link` 평문 제외).
- 비밀 누출 방지: 본 모듈은 `link` / `pass` 를 반환 객체에 포함하되, `describeCodesignConfig`
  를 거치지 않고 외부로 노출하지 않는다 (호출자 책임).

### 3.2 signtool wrapper (PowerShell)

[desktop/scripts/sign-windows.ps1](../../../desktop/scripts/sign-windows.ps1) (신규)

- `param([Parameter(Mandatory)] $Path, [string]$TimestampUrl, [string]$PublisherName, [switch]$Verify, [switch]$AllowSelfSigned)`.
- signtool.exe 자동 탐색 (`ProgramFiles(x86)\Windows Kits\10\bin\x64\signtool.exe` 등).
- 서명 모드:
  - `WIN_CSC_LINK` 가 `https://` 면 임시 `.pfx` 로 다운로드 후 finally 블록에서 삭제.
  - `WIN_CSC_LINK` 가 `http://` 면 거부 (`exit 1`).
  - `signtool sign /fd SHA256 /td SHA256 /tr <ts> /f <pfx> /p <pass> <Path>` 호출.
  - 패스프레이즈는 `$env:WIN_CSC_KEY_PASSWORD` 에서만 읽고 finally 에서 `Remove-Variable`.
- 검증 모드 (`-Verify`):
  - `signtool verify /pa /v <Path>` — chain + timestamp + algorithm 강제.
  - `-AllowSelfSigned` 지정 시 `/pa` 제외 (운영자 빌드 한정).
- 항상 verify 단계로 마무리 (signed 산출물 강제 검증).

### 3.3 평문 비밀 lint

[desktop/scripts/check-codesign-secrets.mjs](../../../desktop/scripts/check-codesign-secrets.mjs) (신규)

- `git ls-files` 트리 한정 스캔 (untracked / node_modules 자동 제외).
- 패턴:
  - `.pfx` / `.p12` 바이너리 — 즉시 차단.
  - `WIN_CSC_KEY_PASSWORD=<value>` 평문 할당 — `$` / `${env.*}` / 빈 값 / 주석은 통과.
  - PKCS#12 base64 헤더 `MII[A-Za-z0-9+/]{40,}`.
  - PEM `-----BEGIN (?:ENCRYPTED )?PRIVATE KEY-----`.
- 예외: 본 lint 자신 / 운영지침 / codesign.ts/.test.ts / sign-windows.ps1 / electron-builder.yml
  (env 변수명만 등장).
- exit 1 on hit — CI publish 워크플로의 `Lint codesign secrets` 단계에서 차단.

### 4.3 신규 회귀 — codesign.test.ts

`desktop/__tests__/codesign.test.ts` — 14 leaf tests, 2 describes:

**`parseCodesignConfig (WIN-073)`** (10 tests)

- env 모두 비어있음 → `enabled: false` + 경고 메시지.
- `WIN_CSC_LINK` + `WIN_CSC_KEY_PASSWORD` 정상 → `enabled: true`.
- `http://` link 거부.
- `file://` link 거부.
- `https://` link 통과.
- pass 누락 → `enabled: false` + "WIN_CSC_KEY_PASSWORD" 경고.
- publisher/subject env 전달.
- timestamp invalid (file://) → 기본값 폴백.
- https timestamp 정상.
- `WIN_CSC_VERIFY_UPDATE_SIGNATURE` 기본 true / 명시 `0` → false.
- `WIN_CSC_ALLOW_SELF_SIGNED` truthy 처리 (`1` ON, `0` OFF).

**`describeCodesignConfig (WIN-073)`** (4 tests)

- enabled 설정의 `pass` / `link` 평문이 결과 문자열에 포함되지 않음 (보안 회귀).
- disabled 설정도 안전 직렬화.
