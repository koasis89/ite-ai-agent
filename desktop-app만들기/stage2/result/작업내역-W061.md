## 2. 목표 (DoD 요약)

- 채팅 영역에 파일을 드래그 시 시각 피드백을 표시하고, 화이트리스트(20종)·단일 50MB·누적 200MB·최대 10건 정책으로 1차 검증해 미확정(pending) 첨부 칩 UI 를 렌더링한다.
- 정책 위반 / `text/uri-list`·`text/html` 등 비-파일 드롭은 항목을 추가하지 않고 채팅 패널에 `system` 역할 메시지로 사유를 안내한다.
- 클립보드 이미지를 `Ctrl+V` 로 paste 하면 `paste-<ts>.<ext>` 이름으로 자동 등록되고, 텍스트 paste 와 충돌하지 않는다.
- 본 티켓은 **렌더러 전용 pending 단계**까지 — IPC/디스크 저장은 WIN-062 에서 이어진다.

## 4. 핵심 API · 정책

### 4.1 화이트리스트 (20종)

```
txt md json csv log png jpg jpeg gif webp svg pdf yaml yml ts js mjs cjs html xml
```

### 4.2 상한

| 항목 | 값 | 상수 |
|---|---|---|
| 단일 파일 | 50MB | `DEFAULT_MAX_FILE_BYTES` |
| 누적 총량 | 200MB | `DEFAULT_MAX_TOTAL_BYTES` |
| 동시 첨부 수 | 10 | `DEFAULT_MAX_FILES` |

### 4.3 거부 사유 (`AttachmentRejection.reason`)

- `extension-not-allowed` · `size-exceeded` · `total-exceeded` · `count-exceeded`
- `empty-name` · `empty-size` · `non-file`

### 4.4 데이터 흐름

```
DragEvent / ClipboardEvent
    │ DropZone
    ▼
{name, size, mime, sourcePath?, blob?}
    │ AttachmentStore.tryAdd
    │   └─ validateAttachmentCandidate (순수)
    ▼
PendingAttachment | null  ──rejected──▶ AttachmentStoreChange.kind='rejected'
                                              └─▶ formatRejection ─▶ chatPanel.viewModel().append('system', …)
```

- Electron `File.path` 는 sourcePath 로 보관, paste 이미지는 `sourcePath=null` + `blob` 으로만 보관 (외부 노출 없음).

## 5. 검증

### 5.1 컴파일/빌드

```
npm run build:desktop
```

→ `tsc -p tsconfig.desktop.json` 성공, `attachments.css` 가 `dist-desktop/desktop/renderer/` 로 복사됨 (1줄 추가 로그 확인).

초기 오류 1건 발견·수정:
- `interface ElectronFile extends File { readonly path?: string }` → DOM `File.path` (non-optional string) 와 호환 불일치(`TS2430`). `readonly path: string` 으로 변경하고 호출부에서 `typeof file.path === 'string' && file.path.length > 0` 로 가드.

## 6. 후속 / 메모

- WIN-062: `AttachmentStore.tryAdd()` 가 반환한 `PendingAttachment` 를 main 측 `attachment_register` IPC 로 승격하는 경계를 잇는다. `sourcePath` 가 있으면 path-only 흐름, `blob` 만 있는 paste 이미지는 임시 디스크 또는 base64 transfer 로 분기 예정(WIN-062 설계).
- WIN-064: 칩 → 메시지 본문 인라인 (`@id` 토큰화 / sha256 표기) 시점에 칩 UI 의 클릭/Enter 키보드 트리거를 활용.
- 추후: drag image preview thumbnail, drop 영역의 ARIA `aria-busy` 토글, 화이트리스트 사용자 설정.
