## 2. 목표

사용자가 `stable` / `beta` 채널을 토글로 선택할 수 있게 한다. 운영자 환경 변수
(`OMX_DESKTOP_UPDATE_ALPHA_ALLOWED=1`) 에서는 `alpha` 채널도 노출된다. 채널 변경 시
즉시 직전 채널의 미적용 다운로드를 폐기하고 (`UpdaterService.discardPendingDownload()`),
다음 `update_check` 부터는 해당 채널의 매니페스트(`latest.yml` / `beta.yml` / `alpha.yml`)
를 조회한다. 모든 입력은 zod enum + ChannelStore 의 화이트리스트 검사로 Defense-in-Depth.

## 3. 구현

### 3.1 채널 SSOT — `desktop/main/updater/channel.ts` (신규)

- `UPDATE_CHANNELS = ["stable","beta","alpha"] as const` + `UpdateChannel` 유니온 타입.
- `isUpdateChannel(v)` / `coerceUpdateChannel(v)` — 화이트리스트 가드 + 안전 폴백.
- `getAvailableChannels(env)` — `OMX_DESKTOP_UPDATE_ALPHA_ALLOWED` 가 `1`/`true` 일 때만 alpha 포함.
- `class ChannelStore` — `current` / `available` getter, `async set(next)` 메서드.
  - 영속 어댑터(`ChannelPersistAdapter`) 주입 가능 (현재는 인메모리만, ConfigStore v2 마이그레이션은 후속 티켓).
  - 변경 시 `onChannelChanged(next, prev)` 콜백 발화 → `UpdaterService.discardPendingDownload()` 트리거.
  - 결과는 `{ok, channel, reason?}`. reason: `invalid-channel` / `no-change` / `alpha-disallowed` / `persist-failed`.

### 3.2 UpdaterService 채널 폐기 — `desktop/main/updater/updater.ts` (수정)

- `discardPendingDownload()` 메서드 추가.
  - 진행 중 backoff `retryHandle` cancel, `retries=0` 리셋.
  - 상태가 `idle` 가 아니면 `transition({state:"idle"})`.
  - `disposed` 인 경우 no-op (멱등).

### 3.3 IPC — `desktop/ipc/commands.ts` (수정)

- `allowedCommands` 에 `update_channel_get`, `update_channel_set` 추가 (33종).
- `UpdateChannelGetArgsSchema = z.array(z.string()).max(0)`.
- `UpdateChannelSetArgsSchema = z.tuple([z.enum(["stable","beta","alpha"])])`.
- `UpdateChannelBackend` 인터페이스 + `setUpdateChannelBackend` / `getUpdateChannelBackend`.
- 백엔드 미초기화 시 폴백:
  - `update_channel_get` → `{current:"stable", available:["stable","beta"], disabled:true}`.
  - `update_channel_set` → `{ok:false, reason:"channel backend not initialised"}`.

### 3.4 부트스트랩 와이어 — `desktop/main/index.ts` (수정)

- `bootstrapUpdater` 가 `parseUpdaterConfig` 직후 `ChannelStore` 인스턴스화.
- `onChannelChanged` 가 `updaterService?.discardPendingDownload()` 호출.
- `setUpdateChannelBackend({getChannel, setChannel})` 등록 — provider=none 환경에서도 read-only 로 동작.
- 실패 경로(`catch`)에서도 `setUpdateChannelBackend(null)` 정리.

### 3.5 Renderer UI — `desktop/renderer/settings/UpdateSettingsPanel.ts` (신규)

- DOM-light 스탠드얼론 클래스 (`mount(host, ipc)`).
- 채널 select + 상태 표시 (`aria-live="polite"`).
- `beta` / `alpha` 선택 시 확인 다이얼로그 (불안정 경고) 호출 — 호스트가 `ipc.confirm()` 구현.
- 응답 reason 별 사용자 메시지 변환 (no-change / persist-failed / error).

### 3.6 패키징 메타 — `electron-builder.yml` (수정)

- `publish.channel: latest` 명시 (stable 빌드 기본값).
- 워크플로 stub 분기 가이드 주석: `v*-beta` 태그 시 `channel: beta` 로 override.

### 4.3 신규 회귀 케이스

- `update-channel.test.ts` (2개 describe, 12개 leaf):
  - UPDATE_CHANNELS 순서 / isUpdateChannel 가드 / coerceUpdateChannel 폴백.
  - getAvailableChannels: env 미설정 시 alpha 제외, `1`/`true` 시 alpha 포함, `0`/`false` 시 제외.
  - ChannelStore 초기화: defaultChannel / 잘못된 default → stable 폴백 / persist.load 우선.
  - ChannelStore.set: 잘못된 입력 거부, 동일 채널 no-change, alpha 미허용 차단,
    영속 false → persist-failed, 영속 throw 스왈로우 + 로그, 콜백 throw 스왈로우 + 로그.
- `ipc-contract.test.ts` › `update_channel_* commands (WIN-072)` (7개):
  - 백엔드 null 시 폴백 — get 은 disabled, set 은 ok=false.
  - 정상 백엔드 — get 스냅샷, set 전환, 동일 채널 시 changed=false.
  - 거부 — 비-enum, 비어있지 않은 args (get), 빈 args (set).

### 4.4 보안 회귀

- IPC zod 가 `"rolling"` 같은 임의 문자열을 `INVALID_REQUEST` 로 차단 (1차 방어).
- `ChannelStore.set` 가 동일하게 `invalid-channel` 로 거부 (2차 방어 — Defense-in-Depth).
- alpha 채널은 `OMX_DESKTOP_UPDATE_ALPHA_ALLOWED` 미설정 환경에서 available 배열에 포함되지
  않아 UI 노출이 차단됨 + setChannel 호출도 `alpha-disallowed` 로 차단.
