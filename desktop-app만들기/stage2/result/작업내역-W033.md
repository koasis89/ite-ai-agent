## 4. 슬래시 매핑 (14종 + /help)

main 측 `allowedCommands` 15종 중 `question_demo` (데모 전용) 를 제외한 14종을 슬래시로 노출.

| 슬래시 | IPC command | 인자 |
|---|---|---|
| `/state` | `state_get_status` | none |
| `/state field <name>` | `state_get_field` | 1 |
| `/hud` | `hud_get_snapshot` | none |
| `/sidecar [team]` | `sidecar_get_snapshot` | 0~1 |
| `/versions` | `versions_get` | none |
| `/platform` | `platform_get` | none |
| `/history [N]` | `history_list` | 0~1 |
| `/bus` | `event_bus_stats` | none |
| `/ask "<질문>" [opt1] … [opt5]` | `question_ask` | 1~6 |
| `/echo <args...>` | `noop_echo` | 0~10 |
| `/sleep <ms>` | `noop_sleep` | 1 |
| `/omx doctor` | `omx_doctor` | none |
| `/omx adapt probe` | `omx_adapt_probe` | none |
| `/omx state` | `omx_state_status` | none |
| `/help` | — | 슬래시 안내 출력 (IPC 미호출) |

### 4.1 매칭 우선순위

`SLASH_COMMANDS_SORTED` 가 `pattern.length` 내림차순으로 정렬되어:
- `/state field platform` → `state_get_field` (먼저 매칭) — `/state` 로 폴백되지 않음.
- `/omx adapt probe` → `omx_adapt_probe` — `/omx state` 또는 `/omx doctor` 와 충돌 없음.

### 4.2 보안/안전망

| 항목 | 처리 |
|---|---|
| 입력 길이 상한 | `MAX_INPUT_LEN = 64KB` → `RangeError` |
| 토큰 수 상한 | `MAX_TOKENS = 32` → `RangeError` |
| 미완료 따옴표 | `tokenizeSlashInput` throw → `emitSystem('슬래시 파싱 오류: …')` |
| 미매핑 슬래시 | `emitSystem('Unknown command: /foo')`, **IPC 미호출** |
| 인자 부족/초과 | `planSlashInvocation` throw → `emitSystem('인자 오류: …')`, **IPC 미호출** |
| dispatcher 예외 | `try/catch` → `emitSystem('IPC 호출 실패: …')` |
| 실제 명령 검증 | main 측 `allowedCommands` + zod (라우터는 SSOT 미복제) |

## 6. 후속 작업

- **매뉴얼 §4.1 갱신**: 사용자 정책상 본 티켓에서 매뉴얼 편집 안 함. 일괄 갱신 시점에 다음 표 추가 필요:
  - 14종 슬래시 + `/help` 와 대응 IPC command
  - 따옴표 그룹 사용 예 (`/ask "긴 질문?" yes no`)
  - 미매핑 처리/인자 오류 메시지 형식
- **WIN-034 (응답 스트리밍)**: 본 티켓이 도구 호출을 `tool` role 단발 메시지로 출력. WIN-034 에서 long-running command 의 progress/partial 이벤트를 같은 메시지에 streaming append 하도록 확장 예정.
- **Phase 6 도구 호출 카드**: 현재 markdown 코드블록(JSON) 으로 결과 표시. 향후 expandable 카드 UI로 일반화.
