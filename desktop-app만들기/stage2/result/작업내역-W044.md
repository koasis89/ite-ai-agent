## 2. 목표

기존 CLI 사용자가 보유한 `.omx/state/*` 디렉터리를 데스크탑 앱에서 **read-only** 로 안전하게 인식하고, 데스크탑 앱의 쓰기 경로는 `%APPDATA%/oh-my-codex/` 1곳으로 명확히 분리한다. 명령 `omx_state_status` 응답에 `legacy.items[].source = 'legacy'` 와 `data.source = 'desktop'` 구분을 추가해 UI 가 두 출처를 분리 표시할 수 있게 한다.

## 4. 핵심 API · 정책

### resolveLegacyStateDir
- 환경 변수 / cwd / userData 의 3단계 fallback
- 모든 후보는 디렉터리 존재 확인 후 `realpathSync` 로 정규화
- 어디에도 없으면 `null` 반환 (legacy 없음 정상 처리)

### LegacyStateAdapter (read-only)
- `readState(name)`: 이름 검증 → `realpathSync` → root prefix 검사 → JSON.parse → `LegacyStateItem` 반환 (실패는 null, 탈출은 throw)
- `listActiveStates()`: flat *.json 만 ASC 정렬, 파싱/탈출 실패 skip, throw 하지 않음
- `writeState`/`deleteState`: 항상 `LegacyStateReadOnlyError` throw
- 데스크탑 앱의 신규 상태는 `%APPDATA%/oh-my-codex/state/` 로 일원화 (legacy 경로는 절대 쓰기 대상 아님)

### `omx_state_status` 응답 확장
```ts
{
  command: 'omx_state_status',
  argv, exitCode, signal, durationMs, stdout, stderr, stdoutTruncated, stderrTruncated,
  source: 'desktop',
  legacy: {
    source: 'env'|'cwd'|'userdata-link',
    rootDir: string,
    readonly: true,
    items: Array<{ name, relativePath, value, source: 'legacy' }>,
  } | null,
}
```
- 기존 필드는 무변 → ipc-contract 회귀 통과
- `legacy === null` 인 경우는 legacy 미발견 (정상)

### `allowedCwdRoots` 정책
- `LocalProcessTransport` / `PtyLocalTransport` 의 `allowedCwdRoots = [process.cwd(), userData]` 유지
- legacy 경로는 **추가하지 않음** — read-only intent 이므로 transport 의 cwd 로 사용될 수 없도록 차단
- 주석으로 정책 명시
