## 2. 목표 (티켓 인용)

- main → renderer `command.progress` 이벤트를 채팅 버블 streaming 토큰 추가에 정식 연결.
- 슬래시 명령(특히 `omx_*`) 실행 시 stdout/stderr 가 채팅 버블에 점진 표시.
- 종료 후 푸터에 exitCode + duration 표시.
- 8KB 초과 chunk 는 truncated 플래그로 잘림 표시.

## 4. 핵심 API 요약

### 4.1 IPC 페이로드 확장 (`command.progress`)

```ts
interface CommandProgressDetail {
  stream: string;             // 'stdout' | 'stderr' | 'pane' | 'info'
  chunk: string;
  messageId?: string;         // WIN-034 — bridge 키
  truncated?: boolean;        // WIN-034 — 8KB 상한 초과 표시
}
```

### 4.2 RunCommand 확장

```ts
RunCommandRequestSchema = z.object({
  command: z.string().min(1),
  args: z.array(z.string()).max(10).optional().default([]),
  messageId: z.string().min(1).max(128).optional(),
});
```

### 4.3 main 측 streaming 어댑터

`runOmxSubcommand(args, { onChunk })` — child stdout/stderr `data` 이벤트마다 누적·cap 처리 후 `onChunk(stream, kept, truncated)` 호출. omx_* 케이스에서 이를 zod-validated `messageId` 와 묶어 `eventBus.publish('command.progress', …)` 발행. 8KB 상한 초과분도 `truncated=true` 로 bridge 에 통보.

### 4.4 renderer StreamingBridge 동작

| 입력 이벤트 | 동작 |
|---|---|
| `command.progress` + messageId | 첫 도착 시 `vm.streamStart('tool')` lazy 생성 → 16ms batch 윈도우에 chunk 누적 → 윈도우 만료 시 `vm.streamAppend(msgId, joined)` |
| `command.progress` + truncated=true | entry.truncated 마킹 (최종 finalize 단계에 표식 추가) |
| `command.completed` + messageId | 잔여 flush + truncated 면 `…(잘림 — 8KB 상한 초과)` append + `*— 종료 exitCode=N duration=Nms*` footer append + `streamEnd` |
| `command.failed` + messageId | 잔여 flush + truncated 표식 + `*— 실패 exitCode=N <message>*` footer + `streamEnd` |
| messageId 미존재 | 무시 — non-streaming 명령(예: `/state`, `/hud`) 은 영향 없음 |
| 다른 type (hud.update 등) | 무시 |

batch 윈도우: 기본 16ms(약 60Hz) — 짧은 입력은 그대로, 폭주 시 윈도우 단위로 묶어 ChatViewModel notify 횟수를 감소시켜 백 압력 완화.

## 6. 후속 작업

- **매뉴얼 §5 갱신**: 사용자 정책상 본 티켓에서 매뉴얼 편집 제외. 일괄 갱신 시점에 추가 필요:
  - StreamingBridge 동작 (progress → tool bubble streaming, 푸터 표시)
  - 8KB 상한 / 잘림 표식 / 16ms batch 윈도우
- **streaming 명령 확장**: 현재 omx_doctor/omx_adapt_probe/omx_state_status 3종만 progress 발행. 향후 `LocalProcessTransport`/`PtyLocalTransport` 기반 일반 워커 명령으로 확장 시 동일 패턴(`onChunk → eventBus.publish('command.progress', { messageId, … })`) 적용.
- **slash 결과 중복 정리**: 현재 streaming bubble + slash-router 의 최종 `formatDispatchResult` tool 메시지가 둘 다 표시됨. 사용자 검토 후 streaming 활성 명령에서는 final summary 를 footer 만으로 대체하도록 후속 티켓에서 결정.
- **Phase 6 도구 호출 카드**: streaming bubble 을 expandable card UI 로 일반화.
