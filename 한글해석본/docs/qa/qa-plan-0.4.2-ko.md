# QA 계획 — v0.4.2(개발자 대 배포된 기본)

날짜: 2026-02-18
범위: `origin/main`(배포됨, `v0.4.1`)과 현재 `dev` 사이의 델타를 검증합니다.

## 1) 패리티 검증 요약

달리다:

```bash
git fetch origin --prune
git rev-list --left-right --count origin/main...dev
git log --oneline origin/main..dev
git diff --name-status origin/main...dev
```

글을 쓰는 시점에서 예상되는 내용:
- `origin/main`은(는) `v0.4.1`에 배포됩니다.
- `dev`은(는) 기능/수정/테스트 커밋(자동 넛지 개선, 모든 작업자 유휴 리더 알림, tmux 입력/스크롤 수정 및 적용 범위 확장)을 앞두고 있습니다.
- `dev`에서 누락된 `main`에서 비병합 기능 커밋이 발견되지 않았습니다.

## 4) 수동 QA 체크리스트

### A. 모든 근로자 유휴 알림
- 2명 이상의 작업자로 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 세션을 시작합니다.
- 모든 작업자가 `idle` 또는 `done`로 전환되도록 합니다.
- 리더가 하나의 유휴 요약 프롬프트를 수신하는지 확인합니다.
- 대기 시간 내에 다시 트리거: 중복된 알림 스팸이 없는지 확인하세요.
- 이벤트/로그 항목이 내보내졌는지 확인합니다.

### B. 자동 너지 패턴 감지
- 새로운 문구(예: "말해", "다음에 할 수 있는 것", "계속 운전")가 포함된 피드 출력입니다.
- 마지막 라인 핫 존에서 지연 감지 트리거를 확인합니다.
- 관련 없는 텍스트가 거짓 트리거를 유발하지 않는지 확인하세요.

### C. Tmux 입력 신뢰성 + 마우스 스크롤
- team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 모드에서 마우스 휠 스크롤 창 기록을 확인합니다.
- CLI 입력 기록에 대해 화살표 키가 계속 작동하는지 확인합니다.
- 작업자에게 반복적으로 메시지를 보내고 제출 일관성을 확인합니다.
- `OMX_TEAM_MOUSE=0`을 설정하고, 세션을 다시 시작한 후, 마우스 모드가 강제로 활성화되지 않았는지 확인하세요.

### D. 구성 생성기 마이그레이션
- 새로운 구성과 기존 구성에서 설정/생성기 경로를 실행합니다.
- `[features]`에 `multi_agent = true` 및 `child_agents_md = true`이 포함되어 있는지 확인하세요.
- 더 이상 사용되지 않는 `collab` 키가 다시 도입되지 않았는지 확인하세요.

### E. `/exit` 프로세스 종료
- `omx`을 실행하고 `/exit`을 호출합니다.
- 프로세스가 정지되지 않고 깔끔하게 종료되는지 확인합니다.
