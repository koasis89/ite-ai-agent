오마이코덱스

OpenAI Codex CLI를 위한 다중 에이전트 오케스트레이션.

npm install -g oh-my-codex

실제 글로벌 버전 범프는 이제 자동 실행 대신 명시적인 알림을 인쇄합니다.omx setup. 준비가 되면 달려가세요omx setup수동으로 또는 사용omx updatenpm을 확인한 다음 동일한 설정 새로 고침 경로를 실행하세요. 새로운 OMX 관리gpt-5.5구성 추천250000 / 200000해당 키가 누락된 경우에만.

0.6.0의 새로운 기능

혼합 코덱스 + 클로드 팀원

다음을 사용하여 팀 워커 라우팅(Routing)OMX_TEAM_WORKER_CLI_MAP(예를 들어:codex,codex,claude,claude) 그래서 하나의 $team 실행에서는 두 CLI를 모두 사용할 수 있습니다.

클로드 팀 리더가 넛지

작업자 측 Codex 후크를 사용할 수 없는 경우에도 알림이 계속 실행되도록 리더 측 모든 작업자 유휴 폴백을 추가했습니다.

더 안전한 트리거 재시도

이제 팀 트리거 폴백은 더 안전한 명확한 재전송 동작을 통해 준비 프롬프트 및 비활성 작업 확인 뒤에 적응형 재전송을 게이트합니다.

안정성 수정

향상된 작업 청구 부트스트랩, 더욱 엄격한 CLI 맵 검증, 예측 가능auto혼합/Claude 팀을 위한 더 나은 리더 창 타겟팅.

핵심 기능

에이전트 프롬프트

구현, 검토, 설계 및 계획을 위한 전문 역할입니다.

스킬

자동 조종 장치 및 팀 파이프라인부터 설정, 릴리스, 추적까지 구성 가능한 워크플로입니다.

팀 오케스트레이션

검증 및 수정 루프가 포함된 단계적 실행 파이프라인.

MCP 상태 관리

세션 전반에 걸쳐 수명 주기 상태, 메모장 및 프로젝트 메모리가 지속됩니다.

빠른 시작

  1. npm install -g oh-my-codex
  2. 설치 또는 실제 버전 범프 후 다음을 실행하십시오.omx setup준비가 되면 직접 사용하거나omx updatenpm이 OMX를 먼저 업데이트하도록 하려는 경우.
  3. omx doctor
  4. omx --xhigh --madmax(신뢰할 수 있는 환경) 또는omx

참조 가이드

  • .omx-config.json 모델/환경 라우팅 참조
  • GitHub/PR/패키지 ID 파이프라인
