# 랄프 취소 계약(규범)

본 계약은 Ralph 취소에 필요한 사후 조건을 정의합니다.
Ralph는 직접 시작되거나 다른 워크플로우 후에 독립형 후속 작업으로 시작되었습니다.

## 필수 사후 조건

Ralph를 취소한 후 구현에서는 다음을 확인해야 합니다.

1. 대상 Ralph 상태는 최종적이고 비활성입니다.
   - `active=false`
   - `current_phase='cancelled'`
   - `completed_at`이 설정되었습니다(ISO8601).
2. 연결 모드 동작:
   - Ralph가 동일한 범위의 Ultrawork/Ecomode에 연결된 경우 해당 연결된 모드도 터미널/비활성이어야 합니다.
   - 동일한 범위 내에서 관련되지 않은 연결 해제 모드는 변경되지 않은 상태로 유지되어야 합니다.
3. 세션 간 안전:
   - 취소는 관련되지 않은 세션의 모드 상태를 변경해서는 안 됩니다.

## 구현 정렬 지점

- `src/cli/index.ts`(`cancelModes`)은 범위 지정 취소 및 연결된 정리 순서를 적용합니다.
- `skills/cancel/SKILL.md`은 범위 인식 취소 동작 및 호환성 대체 정책을 문서화합니다.
