오마이코덱스 홈 시작하기 에이전트(Agent) 스킬(Skill) 통합

시작하기

처음 사용자를 위한 엔드투엔드 설정.

전제조건

  • Node.js 20+ 및 npm.
  • OpenAI Codex CLI가 설치 및 인증되었습니다.
  • 프로젝트 워크플로를 위한 Git(권장)

설치

npm install -g @openai/codex oh-my-codex
omx doctor

언제oh-my-codex실제로 최신 버전과 부딪히면 이제 전역 npm 설치가 시작하는 대신 명시적인 알림을 인쇄합니다.omx setup자동으로. 준비가 되면 다시 실행하세요.omx setup수동으로 또는 사용omx updatenpm을 확인한 다음 동일한 설정 새로 고침 경로를 실행하세요. 새로운 OMX 관리gpt-5.5이제 구성을 권장합니다model_context_window = 250000그리고model_auto_compact_token_limit = 200000해당 키가 누락된 경우에만.

omx doctor설치 형태를 확인합니다. 런타임을 준비된 것으로 처리하기 전에 활성 Codex 프로필이 실제 요청을 수행할 수 있는지도 증명하십시오.

codex login status
omx exec --skip-git-repo-check -C . "Reply with exactly OMX-EXEC-OK"

권장 출시

omx --xhigh --madmax   # trusted environments
omx                    # standard launch

첫 번째 실행 연습

  1. 프로젝트 디렉터리를 생성하거나 입력합니다.
  2. OMX를 전역적으로 설치하거나 업데이트합니다.
  3. 설치 또는 실제 OMX 버전 충돌 후 다시 실행omx setup준비가 되면 직접 사용하거나omx updatenpm이 OMX를 먼저 업데이트하도록 하려는 경우.
  4. 실행omx doctor, 그런 다음codex login status...을 더한omx exec위의 연기 테스트.
  5. 다음으로 Codex CLI를 시작하세요.omx.
  6. 사용$deep-interview "clarify the auth change"의도나 경계가 아직 명확하지 않은 경우.
  7. 사용$ralplan "approve the auth plan and review tradeoffs"구현 계획을 잠급니다.
  8. 선택하다$ralph "carry the approved plan to completion"또는$team 3:executor "execute the approved plan in parallel"실행 스타일을 기반으로 합니다.

프롬프트 지침 소스

기본적으로 OMX 레이어 프로젝트 지침(Codex 동작을 확장하고 Codex 핵심 시스템 정책을 대체/우회하지 않음):

-c model_instructions_file="<cwd>/AGENTS.md"

필요한 경우 동작을 재정의합니다.

OMX_BYPASS_DEFAULT_SYSTEM_PROMPT=0 omx
OMX_MODEL_INSTRUCTIONS_FILE=/path/to/instructions.md omx

문제 해결

문제확인해야 할 사항
omx명령을 찾을 수 없습니다전역 설치를 다시 실행하고 npm 전역 저장소가 PATH에 있는지 확인하세요.
프롬프트를 사용할 수 없습니다.파일이 있는지 확인하세요.~/.codex/prompts/그리고 다시 실행omx setup.
스킬이 로딩되지 않습니다확인하세요:~/.codex/skills/*/SKILL.md파일이 설치되었습니다.
doctor가 구성 문제를 보고함실행omx doctor제안된 수정 사항을 적용합니다.
doctor는 통과했지만 실행은 실패합니다.실행codex login status그리고omx exec --skip-git-repo-check -C . "Reply with exactly OMX-EXEC-OK"동일한 쉘/프로필에서. 그린닥터는 인증된 모델 실행에 대한 증거가 아닌 설치 증거입니다.
사용자 정의 또는 프로필 HOME을 인증할 수 없습니다.활성 상태를 확인하세요.~/.codex또는CODEX_HOME. 컨테이너, 프로필 셸 및 서비스 사용자는 일반 사용자 홈과 동일한 인증/구성을 볼 수 없습니다.
로컬 프록시 또는 호환 엔드포인트가 401을 반환합니다.프록시를 사용하는 경우 활성 상태를 확인하세요.~/.codex/config.toml예상되는 내용이 포함되어 있습니다.openai_base_url. 잘못된 엔드포인트로 전송된 프록시 키는 전달자 인증 누락 또는 잘못된 API 키처럼 보일 수 있습니다.
omx doctor --team죽은 tmux 팀을 본다팀이 의도적으로 포기된 경우 실행omx team shutdown <team-name> --force --confirm-issues, 그 다음에omx cancel, 그 다음에omx doctor --team.
