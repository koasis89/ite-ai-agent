# 릴리스 노트 — 0.14.2

## 요약

`0.14.2`은(는) 빠른 따르기 운영자 신뢰성에 초점을 맞춘 `0.14.1` 이후의 패치 릴리스입니다. 연결된 tmux 창 외부에서 더 안전한 `omx question` 동작, 유휴 후 중복 MCP 형제 자체 정리, 더 엄격한 심층 인터뷰 상태/의도 처리, `ulw` 약어에 대한 한국어 IME 드리프트, `omx question`에 대한 공유 tmux 응답 제출 의미 체계, 더 명확한 심층 인터뷰 배경 질문 지침 및 `dev`에 도착한 TypeScript/Biome 도구 체인 새로 고침.

## 하이라이트

- **`omx question`은 이제 표시되는 tmux 렌더러가 없으면 실패하여 닫히지 않습니다** — 연결된 tmux 창 외부에서 OMX는 이제 분리된 tmux 세션을 자동으로 생성하는 대신 명확한 운영자 측 오류를 반환합니다.
- **중복 MCP 형제는 이제 유휴 후 스스로 정리됩니다** — 오래된 동일 상위 stdio 중복은 안전한 트래픽 후 유휴 기간 후에 자체 종료될 수 있으므로 오래된 MCP 서버 구축이 줄어듭니다.
- **심층 인터뷰 지우기/읽기 동작이 더 안전합니다** — 세션 범위 모드 지우기는 이제 필요할 때 비활성 세션 삭제 표시를 남겨두므로 레거시 루트 폴백 상태가 모드를 즉시 다시 활성화하지 않으며, 실패한 질문은 보류 중인 깊은 인터뷰 의무를 명확하게 시작합니다.
- **한국어 `ulw` 키보드 드리프트가 이제 올바르게 라우팅(Routing)됩니다** — 한국어 2세트 키보드에서 `ㅕㅣㅈ`을 입력한 프롬프트는 키워드 활성화 전에 기존 `ulw` ultrawork 속기로 정규화됩니다.
- **`omx question` 답변 삽입은 공유된 tmux 제출 의미 체계를 재사용합니다** — 질문-답변 삽입은 이제 응답 리스너 창 삽입과 동일한 `buildSendPaneArgvs` 경로를 통과하여 리터럴 텍스트 전달을 유지하고 격리된 `C-m` 제출 및 개행 삭제를 유지합니다.
- **심층 인터뷰 지침은 이제 백그라운드-터미널 격차를 해소합니다** — 제공되는 지침은 에이전트(Agent)에게 인터뷰를 계속하기 전에 백그라운드 `omx question` 터미널이 완료될 때까지 기다리고 JSON 답변을 읽도록 명시적으로 지시하는 반면, 키워드 라우팅은 이제 정리/상태 관리 언급만으로 심층 인터뷰를 활성화하는 것을 방지합니다.
- **도구 기준은 최신입니다** — TypeScript는 `6.0.3`로 새로 고쳐지고, Biome 잠금 파일 메타데이터는 `2.4.12`로 새로 고쳐지며, `tsconfig.json` 핀 TS 6 빌드 경로에 대한 노드 주변 유형입니다.

## 결정된

- 이제 보이지 않는 분리된 렌더러를 생성하는 대신 연결된 tmux 외부의 질문 렌더링이 시각적으로 실패합니다.
- 오래된 중복 MCP 형제는 이제 하나의 부모 아래에 축적되지 않고 안전한 트래픽 후 유휴 후에 자동 종료됩니다.
- 이제 세션 범위 심층 인터뷰 지우기는 레거시 루트 대체 파일이 존재하는 경우에도 지워진 상태를 유지합니다.
- 실패한 심층 인터뷰 질문 시작으로 이제 보류 중인 질문 의무가 지워집니다.
- `ulw` ultrawork 속기에 대한 한국어 IME 오타 활성화.
- `src/question/renderer.ts`과 공유 tmux 창 전송 도우미 간의 질문-답변 제출 드리프트.
- 배경 `omx question` 실행에 대한 심층 인터뷰 프롬프트/템플릿/후크 지침.
- `0.14.2`에 대한 메타데이터 및 잠금 파일 동기화를 해제합니다.

## 남은 위험

- 이는 전체 CI 매트릭스 재실행이 아닌 로컬 릴리스 준비 단계입니다.
- 코드 검토 설계자 파트은 비차단 WATCH 문제를 표시했습니다. 삭제된 세션 삭제 표시 도우미는 CLI 및 MCP 상태 경로 전반에 걸쳐 복제되고, 분리된 질문 렌더러 코드는 유지된 레거시 브랜치로 유지되고, MCP 중복 정리는 여전히 보수적인 프로세스/타이밍 휴리스틱에 의존하며, 심층 인터뷰 지침은 배송된 지침 화면 전체에서 복제된 상태로 유지됩니다.
- 가장 높은 가치의 출시 후 관찰 인터페이스은 여전히 ​​`omx question` 주변의 실제 tmux/재사용 세션 동작, 프롬프트 제출, 워크플로/상태 지우기, 중복 MCP 형제 정리 및 다국어 입력을 통한 워크플로 활성화입니다.
