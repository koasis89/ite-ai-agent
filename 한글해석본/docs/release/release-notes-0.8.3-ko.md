# 오마이코덱스 v0.8.3

출시일: **2026-03-06**

이것은 `0.8.2` 개발 릴리스 라인에 제공되는 Gemini team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 경로에 대한 **집중 핫픽스 릴리스**입니다.

---

## TL;DR

- 첫 번째 명령(`#585`)에 대해 stdin을 사용하는 대신 `--approval-mode yolo -i "<initial inbox prompt>"`을 사용하여 작업자를 시작하여 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 프롬프트 모드에서 Gemini 작업자 시작을 수정합니다.
- 구성된 모델이 명시적으로 Gemini 모델(`#585`)이 아닌 이상 `gpt-5.3-codex-spark`과 같은 Gemini가 아닌 기본 모델이 Gemini 작업자에게 전달되는 것을 방지합니다.
- Gemini 프롬프트 실행 경로(`#585`)에 대한 대상 런타임 및 tmux 세션 회귀 적용 범위를 추가합니다.
- 알림 대체 감시자에 대한 작은 테스트 전용 강화가 포함되어 전체 제품군 릴리스 검증이 부하 시에도 안정적으로 유지됩니다.

---

## 무엇이 바뀌었나

### 1) Gemini 프롬프트 모드 작업자는 이제 명시적인 초기 프롬프트로 시작합니다.

OMX team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 프롬프트 모드를 통해 시작된 Gemini 작업자는 이제 명시적인 초기 지침으로 시작됩니다.

- `--approval-mode yolo`
- `-i "Read and follow the instructions in .../inbox.md"`

**이것이 중요한 이유:**
- Gemini 시작을 위한 stdin 제공 부트스트랩 텍스트에 대한 의존성을 제거합니다.
- 프롬프트 모드에서 Gemini CLI 기대치에 맞게 작업자 부트스트랩을 코디네이션(Coordination)합니다.
- 핫픽스 PR에 보고된 손상된 작업자 불러오기 경로를 수정합니다.

### 2) Gemini 작업자에 대해 Gemini가 아닌 기본 모델 통과가 필터링됩니다.

Gemini 작업자는 더 이상 실수로 Gemini가 아닌 기본 모델을 상속받지 않습니다.

이번 릴리스의 현재 동작:
- 명시적인 Gemini 모델은 여전히 ​​통과합니다.
- Gemini 작업자에 대해서는 Gemini가 아닌 기본값이 생략됩니다.
- 혼합 공급자 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 구성은 잘못된 시작 인수 조합을 방지합니다.

**이것이 중요한 이유:**
- 작업자 시작 중 잘못된 공급자/모델 페어링을 방지합니다.
- 더 깔끔한 혼합 공급자 CLI 상호 운용성을 유지합니다.
- 프롬프트 모드 및 매핑된 작업자 설정에서 예상치 못한 실패를 줄입니다.

### 3) 핫픽스 경로에 대한 회귀 적용 범위가 확장되었습니다.

이 릴리스에는 다음을 다루는 집중 테스트가 추가되었습니다.
- 프롬프트 모드 Gemini 시작 인수 구성
- 프롬프트 실행 Gemini 작업자의 런타임 시작 동작
- 기본 모델이 Gemini가 아닐 때의 변환 동작

### 4) Full Suite 검증이 안정화되었습니다.

릴리스 검증은 또한 불안정한 감시자 테스트를 강화하여 전체 제품군이 스트리밍 EOF-tail 동작을 주장하기 전에 감시자가 준비될 때까지 안정적으로 기다립니다.

**이것이 중요한 이유:**
- 과도한 제품군 부하에서도 릴리스 확인을 결정적으로 유지합니다.
- 고정된 수면에 의존하는 대신 의도된 감시자 동작을 보존합니다.
- 제공된 Gemini 런타임 동작은 변경되지 않습니다.
