# 릴리스 노트 — 0.15.1

## 요약

`0.15.1`은 직접/비 tmux 리더 실행 제어, 수동적 읽기 전용 상태 작업, 구체적인 저장소 인식 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) DAG 종속성 재매핑, 설정/플러그인 모드 복구, 감사된 실행 후속 조치 및 런타임/후크 신뢰성 수정에 초점을 맞춘 `0.15.x` 열차용 패치 릴리스입니다.

## 하이라이트

- `omx --direct` 및 `OMX_LAUNCH_POLICY=direct|tmux|detached-tmux|auto`은 리더 실행 정책을 명시적이고 운영자가 제어할 수 있게 만듭니다.
- 상태 읽기/목록/상태 작업은 수동적이며 더 이상 읽기 부작용으로 `.omx/state` 또는 tmux-hook 구성을 초기화하지 않습니다.
- Repo 인식 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) DAG 핸드오프는 이제 작업 생성 후 기호 종속성을 구체적인 작업 ID로 다시 매핑하고 작업자 받은 편지함/부트스트랩 생성 전에 런타임 작업 종속성 필드를 패치합니다.
- 설정/플러그인 모드 경로는 명시적인 선택 사항을 보존하고, 명시적인 레거시 모드를 복원하고, 오래된 레거시 자산을 안전하게 보관하고, 플러그인 마켓플레이스 검색을 연결합니다.
- 런타임/후크 수정은 표준 중지 수명 주기 읽기, 전송 연결 끊김 후 MCP 상태 지속성, 프롬프트 재개 PID 처리, 후크 진단 오탐 및 macOS 시작 폴링 동작을 개선합니다.

## 호환성

- 기본적으로 지원되는 대화형 실행은 분리된 tmux 관리로 유지됩니다. `omx --direct` 또는 `OMX_LAUNCH_POLICY=direct`을 사용하여 tmux/HUD 관리를 우회하거나 `OMX_LAUNCH_POLICY=tmux|detached-tmux`를 사용하여 tmux 관리 시작을 강제합니다.
- 기존 모델 재정의 및 설정 설치 모드 선택은 그대로 유지됩니다.
- 이 준비 단계에서는 릴리스 태그나 npm 게시가 수행되지 않습니다.
