# 오마이코덱스 v0.8.2

출시일: **2026-03-06**

이는 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 제공자 확장, 더 안전한 기본값, 설정 위생, 설정 전반의 정확성 수정, 키워드 처리 및 OpenClaw 후크 템플릿에 초점을 맞춘 **대상 패치 릴리스**입니다.

---

## TL;DR

- `$team` / team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 런타임은 이제 Codex 및 Claude(`#576`, `#579`, 관련 문제 `#573`)와 함께 **Gemini CLI 작업자**를 시작할 수 있습니다.
- 기본 프론티어 모델 폴백은 이제 하드코딩된 모델 문자열(`#583`) 대신 **`DEFAULT_FRONTIER_MODEL`**을 통해 라우팅(Routing)됩니다.
- 올바른 기술만 제공하는 것에 대해 설정/설치가 더욱 엄격해졌습니다. 이제 **`configure-notifications`** 정식으로 배송되고 `--force`(`#575`, `#580`, `#584`, `#574` 종료)에서 오래된 레거시 스킬(Skill) 디렉터리를 정리합니다.
- `omx setup`은 이제 Codex CLI `>= 0.107.0`(`#572`, `#564` 수정)에 대해 더 이상 사용되지 않는 **`[tui]`** 구성 섹션을 건너뜁니다.
- 해결되지 않은 OpenClaw 자리 표시자(`#581`, `#578` 닫기) 및 키워드 감지 순서/`/prompts` 보호(`#582`)라는 두 가지 추가 패치 수준 버그가 수정되었습니다.

## 무엇이 바뀌었나

### 1) team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 모드: Gemini CLI 작업자 지원

OMX team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 모드는 이제 Codex 및 Claude 외에도 작업자 CLI 공급자로 **Gemini**를 지원합니다.

이 업데이트에 포함된 내용:
- 런타임/세션 확인 시 Gemini 작업자 실행 지원
- Codex/Claude/Gemini 작업자와 혼합 CLI 맵
- `--model` Gemini 작업자를 위한 패스스루 지원
- Gemini 작업자 동작에 대한 확장된 런타임 및 tmux 세션 적용 범위

**이것이 중요한 이유:**
- 혼합 공급자 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션)을 위한 더 많은 유연성
- 공급자별 작업자 역할을 보다 쉽게 ​​실험할 수 있습니다.
- team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 오케스트레이션 인터페이스 전반에 걸쳐 더 나은 패리티

### 2) 이제 모델 대체 기본값이 중앙 집중화되었습니다.

하드코딩된 기본 프론티어 모델 대체 참조가 `DEFAULT_FRONTIER_MODEL`로 대체되었습니다.

이번 릴리스의 현재 동작:
- 기본 프론티어 폴백은 이제 단일 상수를 통해 해결됩니다.
- 해당 상수는 현재 **`gpt-5.5`**로 설정되어 있습니다.
- 복잡성이 낮은 스파크 기본값은 그대로 유지됨 **`gpt-5.3-codex-spark`**

**이것이 중요한 이유:**
- 숨겨진 폴백 불일치 감소
- 더욱 쉬워진 향후 모델 업데이트
- 더욱 깔끔한 테스트 및 구성 의미론

### 3) 설정/설치 동작이 더 깨끗하고 안전해졌습니다.

이제 설치 프로그램은 카탈로그 매니페스트와 현재 Codex 호환성을 더욱 엄격하게 준수합니다.
- `active` / `internal` 기술만 설치합니다.
- 정식으로 `configure-notifications` 배송
- 더 이상 사용되지 않는/병합/별칭 항목을 건너뜁니다.
- `--force` 정리 중에 오래된 배송/레거시 알림 스킬 디렉터리를 제거합니다.
- Codex CLI가 `>= 0.107.0`인 경우 더 이상 사용되지 않는 `[tui]` 섹션 작성을 건너뜁니다.

**이것이 중요한 이유:**
- 더욱 깔끔한 설치 및 업그레이드
- 업그레이드 후 오래된 배송 자산 감소
- 최신 Codex CLI 버전에서는 설정/구성 문제가 줄어듭니다.
- 의사/설정 결과가 혼동될 가능성이 낮습니다.

### 4) 패치 수정

이번 릴리스에는 두 가지 추가 수정 사항이 포함되었습니다.

- **OpenClaw 템플릿 안전성:** 이제 후크 지침 템플릿의 해결되지 않은 자리 표시자가 문자 그대로의 자리 표시자를 지침으로 유출하는 대신 안전하게 대체됩니다(`#581`, `#578` 닫기).
- **키워드 감지 강화:** 명시적인 다중 기술 순서는 왼쪽에서 오른쪽으로 유지되고, 누락된 키워드 별칭은 복원되었으며, 직접 `/prompts:<name>` 호출은 의도하지 않은 암시적 키워드 활성화(`#582`)로부터 보호됩니다.
