## 하이라이트

### 이제 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 시작 시 라우팅(Routing)된 역할 프롬프트를 엔드투엔드 방식으로 준수합니다.

이 핫픽스 릴리스는 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 시작 경로를 완성하므로 라우팅된 작업 역할은 계획 중에 유추될 뿐만 아니라 실제 작업자 시작 지침 화면과 라이브 작업자 메타데이터로 전달됩니다.

이번 릴리스:
- 라우팅된 작업자 역할을 실시간 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 구성 및 작업자 ID로 유지합니다.
- 해결된 역할 프롬프트에서 작업자별 시작 `AGENTS.md` 파일을 작성합니다.
- 명시적인 시작 재정의가 존재하지 않는 한 역할 기반 기본 추론 할당을 활성 상태로 유지합니다.
- 런타임, tmux-session 및 작업자 부트스트랩 적용 범위를 통해 실시간 작업자 시작 경로를 확인합니다.

홍보: [#643](https://github.com/Yeachan-Heo/oh-my-codex/pull/643)

### 확장 작업 부트스트랩은 이제 정식 작업 ID를 유지합니다.

이제 동적 확장은 작업자 부트스트랩 전에 표준 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 상태를 통해 새 작업을 작성하므로 확장된 작업자는 초기 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 시작에 사용되는 런타임 계약과 일치하는 안정적인 작업 ID, 지속적인 역할 및 받은 편지함/작업 메타데이터를 받습니다.

이번 릴리스:
- 종합적인 받은 편지함 전용 작업 메타데이터를 재구성하는 대신 작업자 부트스트랩 전에 확장된 작업을 유지합니다.
- 확장 중에 역할/소유자/작업 ID 충실도를 유지합니다.
- 정식 확장 작업 상태 및 받은 편지함 ID에 대한 회귀 적용 범위를 추가합니다.

프로젝트 범위 OMX 설치를 사용하는 경우 다음을 다시 실행하세요.

```bash

```
