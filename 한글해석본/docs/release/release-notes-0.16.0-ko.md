# 릴리스 노트 — 0.16.0

## 요약

`0.16.0`은 기술 지원 중단 및 기본 Codex 목표 모드 통합에 초점을 맞춘 부 릴리스입니다. 더 이상 사용되지 않는 플러그인 제공 스킬(Skill) 인터페이스을 폐기하고, 장기 실행 작업을 위한 내구성 있는 목표 모드 워크플로를 추가하고, OMX 소유 아티팩트와 Codex 소유 활성 목표 상태 간의 경계를 강화합니다.

이 릴리스는 SHA `e134967863352955feb477e7c4bd2a52b82eeb19` 후보와 함께 `v0.15.3..HEAD`에서 준비되었습니다. 로컬 릴리스 준비에는 커밋되지 않은 메타데이터/문서 업데이트와 준비 상태 차이에 캡처된 검증-안정성 수정 사항도 포함되어 있습니다.

## 하이라이트

- **네이티브 목표 모드 워크플로** — `omx ultragoal`, `omx performance-goal` 및 `omx autoresearch-goal`는 이제 새로운 Codex `get_goal` 스냅샷에 대한 내구성 있는 저장소 아티팩트, 명시적 핸드오프, 검증 원장 및 완료 코디네이션(Coordination)을 제공합니다.
- **잘못된 완료 방지** — 공유 목표 워크플로 도우미는 지속 가능한 완료 체크포인트를 수락하기 전에 예상 목표와 목표 상태를 비교합니다.
- **Ralph와 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 핸드오프 코디네이션(Coordination)** — Ralph 완료 및 승인된 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 실행은 이제 쉘 아티팩트를 숨겨진 Codex 목표 돌연변이로 처리하는 대신 목표 모드 진실 경계를 보존합니다.
- **기술 지원 중단** — 카탈로그 지원이 중단된 설치 가능/플러그인 제공에서 사용되지 않는 스킬이 폐기되었습니다. 더 이상 사용되지 않는 루트 래퍼는 호환성 스텁으로 남아 있을 수 있습니다.
- **자동 조사 마이그레이션 경로** — 직접 `omx autoresearch`은 더 이상 사용되지 않습니다. 목표 모드 지원 연구 워크플로에는 `$autoresearch` 또는 `omx autoresearch-goal`을 사용하세요.
- **운영 강화** — 질문 배치 핸드셰이크, 알림 프록시 지원, 시작 범위 탐색, 박스형 상태 라우팅(Routing), 오래된 중지 처리 및 CI/패키지 템플릿 적용 범위가 전체 릴리스 범위에서 강화되었습니다.

## 목표 모드 업그레이드 참고 사항

`ultragoal`, `performance-goal` 및 `autoresearch-goal`에는 지속적인 완료 코디네이션(Coordination)을 위해 새로운 Codex 목표 스냅샷이 필요합니다. OMX는 숨겨진 Codex 목표 상태를 직접 변경하지 않습니다. 운영자는 이러한 워크플로가 `get_goal`, `create_goal` 및 `update_goal`에 대한 모델 지향 핸드오프를 인쇄한 다음 체크포인트 완료 시 결과로 나오는 새로운 `get_goal` JSON을 요구할 것으로 예상해야 합니다.

## 호환성 / 마이그레이션

- 플러그인 번들은 더 이상 레거시 분할 질문/검토/도우미 항목과 같은 카탈로그에서 더 이상 사용되지 않는 기술 화면을 설치하지 않습니다. 통합 `ask` 워크플로와 목표 모드 워크플로 스킬을 선호합니다.
- 더 이상 사용되지 않는 루트 기술 래퍼는 호환성을 위해 유지될 수 있지만 활성 설치 가능/플러그인 제공 인터페이스으로 처리되어서는 안 됩니다.
- 기존 Team/Ralph 사용자는 Codex 목표가 활성화되면 더 엄격한 완료 동작을 볼 수 있습니다. 완료 주장은 목표 및 상태 증거와 일치해야 합니다.
- 이 로컬 준비 단계에서는 태그, npm 게시 또는 GitHub 릴리스 게시가 수행되지 않습니다.
