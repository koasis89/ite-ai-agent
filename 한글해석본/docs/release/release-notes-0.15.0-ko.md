# 릴리스 노트 — 0.15.0

## 요약

`0.15.0`은 현재 `dev` 열차에 대한 마이너 릴리스를 준비합니다: 자사 플러그인 제공, Codex 앱 호환성, Visual Ralph, 설정 설치 모드 선택, 기본 에이전트(Agent)/모델 라우팅(Routing), 후크/런타임 강화, Windows/tmux 질문 안정성, CI 중단 방지, Rust 호환성 및 릴리스 준비 자료.

범위 참고 사항: `v0.14.4`은 현재 `dev` 상위 항목에 존재하며 이 후보에 대해 유효한 연결 가능한 기반이 아닙니다. 따라서 릴리스 비교 및 ​​준비 상태 증거는 `0.14.4` 릴리스 노트/준비 상태 파일을 기록 자료로 유지하면서 확인된 연결 가능한 기반으로 `v0.14.3`을 사용합니다.

## 하이라이트

- 자사 Codex 플러그인 패키징은 이제 `plugins/oh-my-codex`, 플러그인 설명자, 마켓플레이스 메타데이터, 패키지 레이아웃 테스트 및 미러 동기화 확인을 포함하여 릴리스 인터페이스의 일부입니다.
- Codex 앱 호환성 경로는 tmux 전용 런타임 가정을 피하고 플러그인 접두사가 붙은 기술 라우팅을 유지합니다.
- Visual Ralph는 라우팅, 생성된 문서 및 회귀 적용 범위를 갖춘 최고 수준의 워크플로 스킬(Skill)로 제공됩니다.
- 설치 프로그램은 플러그인 대 레거시 설치 모드를 보존하고, 정리/백업을 명확하게 보고하고, 로컬 선택 사항을 덮어쓰지 않고도 후크/런타임 자산을 정렬된 상태로 유지할 수 있습니다.
- **실행 동작은 기본적으로 명시적이며 운영자가 제어할 수 있습니다.** 기본 실행 경로는 detached-tmux로 유지됩니다. `omx --direct` 또는 `OMX_LAUNCH_POLICY=direct`(또는 `=tmux`/`=detached-tmux`)을 사용하여 원하는 실행 모드를 강제 실행합니다.
- 기본 에이전트 및 모델 라우팅 계약은 정의, 생성된 모델 테이블, 설정 새로 고침 경로 및 런타임 지침 전반에 걸쳐 적용됩니다.
- 후크/런타임 강화에는 Stop-hook 구문 분석 가능성, 알림 폴백 감시자, 파생 감시자, 오래된 tmux 소켓, team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 ID 및 CI 묵음 보호가 포함됩니다.
- Windows/tmux 질문 처리 및 Rust 1.73 호환 탐색 하네스 동작은 대상 테스트에서 다룹니다.

## 호환성

- 기존 레거시 기술 설치에는 사용자 마이그레이션이 필요하지 않습니다.
- 플러그인 모드 설치는 릴리스 소유자가 패키지/릴리스를 게시한 후 번들로 제공되는 자사 플러그인을 사용해야 합니다.
- 기존 모델 재정의는 의미를 유지합니다. 생성된 기본값은 해당 파트에 대해 계속해서 `gpt-5.5`, `gpt-5.4-mini` 및 `gpt-5.3-codex-spark`를 선호합니다.
- 이 준비 단계에서는 릴리스 태그나 npm 게시가 수행되지 않습니다.
