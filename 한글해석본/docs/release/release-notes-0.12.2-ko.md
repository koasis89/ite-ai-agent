# 릴리스 노트 — 0.12.2

## 요약

`0.12.2`은 누적된 `v0.12.1..v0.12.2` 패치 트파트입니다. 기본 Windows + psmux team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 부팅 및 종료 안전, 출시 후 모드-상태 종료-경주 복구, HUD/오버레이/키워드 감지기/MCP 상태 전반에 걸친 정식 HUD 기술 상태 가시성, 모니터 기반 런타임-cli 종료 시 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 상태 보존 및 동기화된 `0.12.2` 릴리스 자료입니다.

## 수정 사항 및 변경 사항이 포함되었습니다.

- 기본 Windows + psmux 분할 창 종료는 리더/클라이언트 조상이 겹칠 때 프로세스 트리 사전 종료를 건너뛰어 리더 창을 유지합니다(#1358).
- `postLaunch` 모드 상태 정리는 이제 비어 있거나 잘린 shutdown-race JSON을 최소 비활성 레코드로 복구하는 반면 구조적으로 완전한 잘못된 JSON은 경고하고 그대로 둡니다(#1360)
- 기본 Windows team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 창은 POSIX `/bin/sh -lc` 대신 env + PATH 부트스트랩을 사용하여 인코딩된 PowerShell 명령을 통해 부팅되므로 작업자는 실제로 준비가 보고됩니다(#1362).
- HUD 워크플로우 가시성은 HUD, 오버레이, 키워드 활성화 및 MCP 상태 전반에 걸쳐 세션 범위 스킬(Skill) 활성 상태를 통해 표준화됩니다. 오래된 루트 전용 모드 배지는 더 이상 세션보다 오래 지속되지 않으며 레거시 단일 기술 리더는 계속 호환됩니다(#1367).
- `runtime-cli`의 모니터링 감지 터미널 및 실패 조건은 보고 전용 경로에 유지되므로 운영자가 명시적으로 종료를 요청할 때까지 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 상태가 보존됩니다(#1369).
- 릴리스 메타데이터 및 보충 자료는 Node, Cargo, 변경 로그, 릴리스 본문 및 릴리스 준비 문서 전체에서 `0.12.2`에 맞춰 정렬됩니다.

## 남은 위험

- 이는 전체 CI 매트릭스 재실행이 아닌 로컬 확인 패스입니다.
- Windows 관련 수정 사항(#1358, #1362)은 라이브 기본 Windows 11 + psmux 연기 실행 없이 제공됩니다. 릴리스 후 모니터링은 기본 Windows team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 작업자 부팅 및 분할 창 종료 원격 측정을 추적해야 합니다.
- HUD 표준화 변경(#1367)은 여러 가시성 인터페이스에 동시에 적용됩니다. 출시 후 모니터링은 레거시 단일 기술 리더 및 다중 워크플로 배지 렌더링의 드리프트를 감시해야 합니다.
