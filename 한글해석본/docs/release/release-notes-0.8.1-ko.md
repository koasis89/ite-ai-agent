# 오마이코덱스 v0.8.1

출시일: **2026-03-05**

이는 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 코디네이션(Coordination) 신뢰성, 알림 설정 단순성 및 보다 안전한 OpenClaw 운영에 초점을 맞춘 **안정성 + 유용성** 릴리스입니다.

---

## TL;DR

`$team`/`omx team`을 많이 사용하는 경우 이 릴리스에서는 **CLI 우선 상호 운용성**을 표준화하여 런타임의 일관성을 높입니다.

알림을 설정하는 경우 이제 **통합된 `configure-notifications` 흐름**을 통해 온보딩이 더 깔끔해졌습니다.

OpenClaw 통합을 사용하는 경우 이제 시간 초과 처리에 **안전하게 구성 가능한 범위**가 있습니다.

---

## 무엇이 바뀌었나

### 1) team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 런타임: 이제 CLI 우선 상호 운용성이 기본 방향입니다.

- 다음을 통해 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) API 상호 운용성을 추가하고 완성했습니다.
  - `omx team api ...`
- 레거시 `team_*` MCP 도구는 이제 CLI 우선 계약을 위해 더 이상 사용되지 않는 경로로 처리됩니다.

**이것이 중요한 이유:**
- team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 코디네이션(Coordination) 흐름 전체에서 보다 예측 가능한 동작
- 작업자/리더 상호 작용을 위한 더욱 깔끔한 호환성 인터페이스
- team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 런타임 계약에 대한 장기 유지 관리 효율성 향상

### 2) 알림: 이제 설정이 통합되었습니다.

- 알림 설정 지침이 단일 워크플로로 리팩터링되었습니다.
  - `configure-notifications`

**이것이 중요한 이유:**
- 단편화된 설정 경로 감소
- 신규 사용자를 위한 더 쉬운 온보딩
- 알림 공급자 간의 구성 드리프트 가능성이 낮습니다.

### 3) OpenClaw 안전성 + 조작성 개선

- OpenClaw 명령 시간 초과는 이제 제한된 안전 제한으로 구성 가능합니다.
- 더 강력한 토큰/명령 안전 지침과 실용적인 개발 런북을 통해 문서가 확장되었습니다.

**이것이 중요한 이유:**
- 자동화가 많은 환경에서 더욱 안전한 작동
- 개발 및 사고 후속 조치를 위한 운영 명확성 향상

## 호환성/마이그레이션 참고사항

- 이전에 레거시 `team_*` MCP 워크플로를 사용했다면 다음으로 마이그레이션하세요.
  - `omx team api <operation> ...`
- 알림 설정의 경우 다음을 선호하세요.
  - `omx configure-notifications`(또는 이에 상응하는 기술)

이번 패치 릴리스에는 주요 패키지 수준 API 변경 사항이 도입되지 않았습니다.
