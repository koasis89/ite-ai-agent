# 릴리스 노트 — 0.14.1

## 요약

`0.14.1`은 심층 인터뷰 질문 적용, `omx question` tmux 렌더링, 업데이트/설정 새로 고침 동작, 중지/수명 주기 정규화, 소규모 지침/CI 강화 등 새로운 대화형 오케스트레이션 인터페이스에 대한 안정성 후속 조치에 중점을 둔 `0.14.0` 이후의 패치 릴리스입니다.

## 하이라이트

- **대기 중인 심층 인터뷰 질문 의무로 인해 이제 올바르게 중지가 차단됩니다**. 모드가 이미 비활성으로 표시된 경우에도 OMX는 여전히 대기 중인 구조화된 질문을 차단 의무로 처리합니다.
- **질문 창은 tmux 환경에서 더욱 탄력적입니다** — POSIX가 아닌 셸, 누락된 창 및 분리된 세션 시작 오류가 이제 자동으로 죽은 질문 UI를 남기는 대신 페일클로즈됩니다.
- **설정 새로 고침은 복구가 더 쉽습니다** — 명시적 `omx update`은 설치된 버전이 이미 최신 버전인 경우에도 설치 상태를 새로 고칠 수 있으며 권고 업데이트 상태 쓰기 오류가 더 이상 새로 고침 경로를 차단하지 않습니다.
- **라이프사이클 호환성 코드가 중앙 집중화됨** — 이제 터미널 라이프사이클 도우미는 병렬 정규화 구현을 수행하는 대신 공유 런타임 실행 결과 계약을 재사용합니다.
- **검토/탐색 지침은 현재 기본값과 일치합니다** — 코드 검토 스킬(Skill)이 더 강력해지고 경량 폴백 파트이 의도적으로 간결하게 유지됩니다.

## 결정된

- 심층 인터뷰 대기 중인 질문 의무에 대해 얽매이지 마세요.
- tmux/Codex 창의 질문 렌더러/세션 활성 및 답변 제출 안정성.
- 설치 후/설정 새로 고침 루팅 및 명시적 업데이트 오래된 설정 재시도 동작.
- Stale Ralph / Skill-Active / UltraWork 세션 간 유출을 중지합니다.
- `0.14.1`에 대한 메타데이터 및 잠금 파일 동기화를 해제합니다.
