# 릴리스 노트 — 0.14.3

## 요약

`0.14.3`은 현재 `dev` 강화 열차에 대한 `0.14.2` 이후의 패치 릴리스입니다: 질문/심층 인터뷰 반환 창 신뢰성, 프로젝트-로컬 탐색 실행 컨텍스트, TOML 복구 안전 설정, tmux HUD 창 대상 지정, ultrawork 프로토콜 정렬, BusyBox 정리 호환성, 오래된 중지/자동 조종 장치 상태 처리, 표준 런타임 감독자 이벤트, Docker-host tmux 질문 렌더링 및 기본 Windows psmux 작업자 창 부트스트랩 경화.

## 하이라이트

- `omx question`은 이제 도구 실행 프롬프트, 프롬프트 재시드, 분리/표시 렌더러 경로 및 렌더러 메타데이터 경합 전반에서 호출 리더 창을 더 잘 보존합니다.
- 심층 인터뷰는 이제 다시 질문하기 전에 답변한 질문 기록을 코디네이션(Coordination)하고 대규모 설명 흐름에 대한 요약 게이트를 추가합니다.
- 지속형 설정 범위에서 프로젝트 로컬 `CODEX_HOME`을 확인하여 탐색 시작 중에 프로젝트 범위 설정이 적용됩니다.
- 설정 새로 고침은 고아 조각 없이 여러 줄 루트 TOML 할당을 처리합니다.
- HUD 코디네이션(Coordination)은 창 간 크기 코디네이션(Coordination) 드리프트를 방지하면서 방출하는 tmux 창을 대상으로 합니다.
- 정리는 해당 호환성 오류에 대해서만 `command` 필드에서 `args`로 대체하여 BusyBox `ps`을 지원합니다.
- 런타임/team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 인터페이스에는 표준 감독자 이벤트와 기본 Windows psmux 창 부트스트랩 강화가 추가됩니다.

## 호환성

- 사용자 마이그레이션이 필요하지 않습니다.
- `omx question`은 표시/반환 가능한 tmux 창이 없을 때 의도적으로 장애가 발생한 상태로 유지됩니다.
- 프로젝트-로컬 `.codex` 사용자는 업데이트/설정 새로 고침 후 보다 일관된 실행 동작을 받아야 합니다.
