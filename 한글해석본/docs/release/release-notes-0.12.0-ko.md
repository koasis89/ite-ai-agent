# 릴리스 노트 — 0.12.0

## 요약

`0.12.0`은(는) `0.11.13` 이후의 부 릴리스입니다. 왜냐하면 제공된 델타가 기본 Codex 후크 동작, team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션)/런타임 전달, Windows/tmux 감독 및 워크플로 문서 전반에 걸쳐 광범위하고 사용자에게 표시되기 때문입니다.

## 수정 사항 및 변경 사항이 포함되었습니다.

- 기본 Codex 후크 소유권은 이제 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션)이 아닌 OMX 세션에 대한 저장소/런타임 계약에 있습니다.
- 자사 Bash `PreToolUse` / `PostToolUse` 지침이 제공되고 문서화되었습니다.
- team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) 런타임 전달, 사서함 처리, 창 상태 가시성 및 다음 작업 코디네이션(Coordination)이 더욱 강력해졌습니다.
- 리더 넛지, 미리 알림, 세션 연속성 및 오류 지속 경로는 실시간 운영자 흐름 전반에 걸쳐 강화됩니다.
- Windows/tmux 실행 안정성 및 명령 해상도가 향상되었습니다.
- 프롬프트 / AGENTS 기본값은 이제 품질 우선, 증거 기반 실행을 강조합니다.
- 번역된 README/문서 자료는 `docs/readme/` 아래 확장 및 재구성됩니다.
- 릴리스 메타데이터 및 자료가 `0.12.0`에 맞춰져 있습니다.

## 이것이 마이너 릴리스인 이유

- `v0.11.13`이 변경된 파일 `185`, 비병합 커밋 `65`, 병합 커밋 `26`에 걸쳐 있으므로 릴리스 트파트
- 패치 수준의 정확성 수정이 아닌 새로운 자사 네이티브 후크 동작을 도입합니다.
- 이는 운영자가 직면하는 문서/지침 및 릴리스/런타임 인체공학을 의미 있게 변경합니다.
