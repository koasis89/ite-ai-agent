# 오마이코덱스 v0.8.4

출시일: **2026-03-06**

이는 `omx setup` 새로 고침 동작을 더욱 안전하고 예측 가능하며 재실행하기 쉽게 만드는 데 초점을 맞춘 **설정 흐름 패치 릴리스**입니다.

---

## TL;DR

- `omx setup`은 이제 오래된 생성 콘텐츠를 남겨두는 대신 기본적으로 관리형 OMX 아티팩트를 새로 고칩니다.
- 관리형 새로 고침 경로는 이제 파일을 덮어쓰기 전에 해당하는 경우 백업을 보존합니다.
- 관리되는 Codex 모델 참조를 `gpt-5.3-codex`에서 `gpt-5.5`로 업그레이드하기 전에 설치 메시지가 표시됩니다.
- 설정 및 구성 생성 경로에 대해 더 깊은 새로 고침/멱등성 회귀 적용 범위를 추가했습니다.
- 소규모 릴리스 검증 강화가 포함됩니다. 감시자 종료 정리 안정성과 엄격한 미사용 게이트에 의해 나타나는 데드 코드 정리가 포함됩니다.

---

## 무엇이 바뀌었나

### 1) 기본적으로 관리형 OMX 아티팩트 새로 고침

이제 설치 프로그램에서는 관리형 OMX 아티팩트를 일회성 삭제가 아닌 새로 고칠 수 있는 출력으로 처리합니다. `omx setup` 업데이트를 다시 실행하면 아티팩트가 더욱 일관되게 제공되어 기존 설치가 현재 템플릿 및 생성된 자산과 일치하도록 돕습니다.

**이것이 중요한 이유:**
- 업그레이드 후 오래된 생성 파일 감소
- 반복 설정 실행을 더욱 안전하고 유용하게 만듭니다.
- 신규 설치와 새로 고침 설치 간의 일관성이 향상됩니다.

### 2) 새로 고침 경로는 덮어쓰기 전에 백업을 보존합니다.

설정이 관리되는 아티팩트를 대체할 때 이제 해당하는 경우 더 강력한 백업 동작을 수행합니다.

**이것이 중요한 이유:**
- 기존 로컬 OMX 관리 파일을 새로 고칠 때 위험이 낮아집니다.
- 이전 상태를 검사해야 하는 경우 사용자에게 더 명확한 복구 경로를 제공합니다.
- 설정 자동화를 덜 파괴적으로 만듭니다.

### 3) 이제 모델 업그레이드가 다시 작성되기 전에 설치 메시지가 표시됩니다.

관리되는 구성 새로 고침으로 Codex 모델 참조가 `gpt-5.3-codex`에서 `gpt-5.5`로 업그레이드되면 이제 설정에서 해당 변경을 수행하기 전에 묻습니다.

**이것이 중요한 이유:**
- 일상적인 새로 고침 중에 예상치 못한 모델 업그레이드를 방지합니다.
- 설치 프로그램이 기존 구성을 수정하려고 할 때 사용자 신뢰를 유지합니다.
- 자동 재작성을 강제하지 않고 관리형 기본값을 최신 상태로 유지합니다.

### 4) 새로 고침 및 멱등성을 위해 확장된 회귀 범위

이 릴리스에서는 다음과 같은 테스트 및 검증 강화를 추가/확장합니다.
- 새로 고침 동작 설정
- 범위가 지정된 덮어쓰기 처리
- 설치 관리 새로 고침 중 호환성 제거
- 구성 생성기 멱등성 및 알림 인식 생성 흐름
- 스트리밍 대체 테스트 중 감시자 종료/정리 동기화
