# 릴리스 노트 — 0.14.0

## 요약

`0.14.0`은 대화형 오케스트레이션 변경에 초점을 맞춘 `0.13.2` 이후의 부 릴리스입니다. `omx question`는 OMX 소유의 차단 질문 인터페이스이 되고, 심층 인터뷰는 이제 구조화된 질문 의무를 통해 실행되고, 자동 연구는 기술 우선 및 검증자 관리가 되고, 키워드가 아닌 프롬프트에 대한 자문 분류 라우팅(Routing) 랜드가 되며, 런타임 실행 결과가 정규화되어 중지/계속 동작이 일관되게 유지됩니다.

## 하이라이트

- **`omx question`은 이제 대화형 질문 인터페이스입니다** — 에이전트(Agent)가 호출한 사용자 질문은 JSON 입력, UI 렌더링, 지속적인 질문 상태 및 구조화된 답변이 포함된 구조화된 OMX 진입점을 통과합니다.
- **심층 인터뷰 상호 작용은 명시적이고 시행 가능합니다** — 이제 각 인터뷰 라운드는 일반 텍스트 대체에 의존하는 대신 질문 의무를 생성/충족합니다.
- **자동 조사는 더욱 엄격하고 안전합니다** — 직접 `omx autoresearch`은 더 이상 사용되지 않으며 완료하려면 최선의 노력보다는 검증인의 증거가 필요합니다.
- **프롬프트 라우팅에는 더 가벼운 권고 파트이 있습니다** — PASS/LIGHT/HEAVY 분류 힌트는 워크플로를 암시적으로 활성화하지 않고도 키워드가 아닌 프롬프트를 코디네이션(Coordination)할 수 있습니다.
- **릴리스 유효성 검사는 개발 작업 공간에서 다시 재현 가능** — Lint는 이제 추적된 소스 루트를 대상으로 하므로 런타임/작업 트리 디렉터리 아래에 중첩된 로컬 Biome 루트가 릴리스 게이팅을 중단하지 않습니다.

## 추가됨

- `omx question` 구조화된 질문 페이로드, UI 실행, 상태 경로 통합 및 JSON 출력에 대한 CLI 지원.
- 생성/만족/지우기 흐름을 위한 심층 인터뷰 질문-의무 상태 전환.
- 자문 분류 휴리스틱 및 지속적인 후속 조치 억제.

## 변경됨

- 이제 심층 인터뷰 라운드에는 대체 임시 프롬프트 대신 `omx question`이 필요합니다.
- Autoresearch CLI 항목은 `$deep-interview --autoresearch` 및 `$autoresearch`을 위해 더 이상 사용되지 않습니다.
- 런타임 실행 결과는 공유된 터미널/비터미널 의미 체계로 정규화됩니다.
- 전문가 라우팅 지침과 역할 라우터 소유권 경계가 더욱 엄격해졌습니다.
- `npm run lint`은 이제 `biome lint src bin`을 실행합니다.

## 결정된

- 이제 게이팅 중지는 대기 중인 대화형 질문 의무와 불완전한 자동 조사 검증을 존중합니다.
- 질문 렌더러/주입 수명 주기는 이제 하나의 소유된 질문 경로를 통해 실행됩니다.
- 릴리스 메타데이터와 잠금 파일은 `0.14.0`에 동기화됩니다.

## 남은 위험

- 이는 여전히 전체 CI 매트릭스 재실행이 아닌 로컬 릴리스 준비 단계입니다.
- 대화형 tmux/UI 동작은 대상 테스트를 통해 잘 다루어지지만 실제 다중 세션 운영자 동작은 출시 후 관찰 인터페이스에서 가장 높은 가치를 유지합니다.
