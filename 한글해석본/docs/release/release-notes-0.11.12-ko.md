# 릴리스 노트 — 0.11.12

## 요약

`0.11.12`은 더 많은 Windows 터미널 깜박임 경로를 제거하고, 추가 team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션)/런타임 이음새 간격을 줄이고, 노드 테스트 실행을 크로스 플랫폼으로 만들고, 현재 OMX 온보딩 경로를 기준으로 워크플로 문서를 정렬하는 `0.11.11` 이후의 패치 릴리스입니다.

## 수정 사항 및 변경 사항이 포함되었습니다.

- Windows 하위 프로세스 실행은 이제 더 넓은 `windowsHide` 적용 범위를 사용합니다.
- git 메타데이터 읽기는 Windows conhost 깜박임을 방지하기 위해 필요한 경우 파일 시스템 읽기로 대체됩니다.
- team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션) cwd 메타데이터 확인은 현재 매니페스트 v2 정보 소스로 표준화되었습니다.
- 발송/사서함 전환으로 런타임 씬 어댑터 이중 쓰기 이음새 간격이 더 많이 줄어듭니다.
- tmux 준비 및 자동 넛지 동작은 OMX 관리 세션으로 유지됩니다.
- 노드 테스트 파일 실행은 이제 shell-`find`에 종속되지 않고 크로스 플랫폼입니다.
- 연결된 레거시 스킬(Skill) 루트는 공유된 정식 루트를 통해 해결됩니다.
- 이제 워크플로 문서는 심층 인터뷰 → ralplan → team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션)/ralph를 통해 사용자에게 일관되게 안내합니다.
- 릴리스 메타데이터는 노드, Cargo 작업 공간 메타데이터 및 잠금 파일 전체에서 `0.11.12`에 맞춰 정렬됩니다.

## 남은 위험

- 이 릴리스 검증은 릴리스 무결성과 `0.11.11` 이후 런타임/테스트/문서 인터페이스을 목표로 합니다. 전체 CI 매트릭스를 다시 실행하는 것은 아닙니다.
- 향후 워크플로 문서 편집은 심층 인터뷰 → ralplan → team(멀티 worker(실제작업을 수행하는 실행단위) 인터페이스 실행 오케스트레이션)/ralph 진행에 고정되어 릴리스 문서가 이전 진입점 지침으로 돌아가지 않도록 해야 합니다.
