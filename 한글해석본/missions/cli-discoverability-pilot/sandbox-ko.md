# 샌드박스 규칙

- CLI 검색 가능성에만 초점을 맞춰 변경 사항을 유지하세요.
- 검토 가능한 가장 작은 차이를 선호합니다.
- 종속성을 추가하지 마십시오.
- 명령 의미를 유지합니다. 이 임무는 기능 재설계가 아니라 발견 가능성에 관한 것입니다.
- 평가자는 대상 빌드 + CLI 검색 가능성 테스트 슬라이스를 통과하여 보상합니다.

# 평가정책

- `pass=true`은 대상 빌드 및 검색 가능성 테스트가 모두 통과되었음을 의미합니다.
- `score`은 `0.00`에서 `1.00`까지 통과한 대상 검사의 비율입니다.
- 점수가 높을수록 좋습니다.
