Ndjson 봉투(Envelope)는 에이전트와 데스크톱 앱 사이의 **데이터 통신 규약**을 의미합니다. 단순히 데이터를 "날것(Raw)"으로 쏘는 것이 아니라, 데이터를 **'봉투'라는 표준 규격 안에 넣어** , JSON 형식으로 한 줄씩 끊어서 보내는 스트리밍 전송 방식입니다.

이 방식이 왜 필요한지, 그리고 왜 이 구조가 강력한지 정리해 드립니다.

### 1. 왜 "Ndjson 봉투"인가?

* **Ndjson (Newline-Delimited JSON):**
* 보통의 JSON 파일은 전체가 하나의 `{...}` 객체로 묶여 있어, 데이터가 다 생성되어야 파싱이 가능합니다.
* 반면, **Ndjson**은 각 JSON 객체가 줄바꿈(`\n`) 단위로 독립적입니다. 즉, 데이터가 생성되는 족족 한 줄씩 실시간으로 전송(Streaming)할 수 있습니다.


* **봉투(Envelope):**
* 데이터 앞에 항상 공통적인 '봉투' 정보를 붙입니다.
* 이 봉투 덕분에 받는 쪽(Electron)은 파싱하기 전에 **"이 데이터가 텍스트 토큰인지, 도구 호출인지, 사용자 질문인지"를 즉시 식별**할 수 있습니다.



### 2. 봉투(Envelope)의 표준 구조

`interop-team-mutation-contract-ko.md` 및 로드맵에서 정의한 규격에 따라 모든 데이터는 아래와 같은 형태를 가집니다.

```json
{
  "schema_version": "1.0",
  "ok": true,
  "type": "token", 
  "data": { "text": "안" }
}

```

* **`schema_version`**: 프로토콜 버전 (하위 호환성 유지)
* **`ok`**: 오퍼레이션 성공 여부
* **`type`**: 데이터의 목적 (핵심!)
* `token`: 클로드 스타일의 타이핑 UX 구현용
* `tool_call`: 백그라운드 작업 수행 시각화용
* `interlude`: 사용자 판단이 필요한 차단형 인터뷰 모드 전환용


* **`data`**: 각 타입별 실제 본문 (가변적)

### 3. 클로드 AI 스타일 UX를 위한 봉투 활용 예시

에이전트가 "파일을 읽고 요약해줘"라는 명령을 수행할 때, 실제 스트림은 이런 식으로 흘러옵니다.

1. **시작 알림**: `{"type": "agent_init", "data": {"persona": "planner"}}`
2. **생각 시작**: `{"type": "token", "subType": "reasoning", "data": {"text": "파일을 "}}`
3. **도구 호출**: `{"type": "tool_call", "data": {"tool": "read-file", "args": "..."}}`
4. **답변 출력**: `{"type": "token", "subType": "content", "data": {"text": "요약 결과..."}}`

### 4. 개발팀 구현 포인트 (Node.js/Electron)

Electron의 Main Process는 이 스트림을 받기 위해 **`readline` 모듈**을 사용합니다. 이게 가장 효율적입니다.

```typescript
const readline = require('readline');

// CLI의 stdout을 한 줄씩 읽는 파이프라인
const rl = readline.createInterface({
  input: childProcess.stdout,
  terminal: false
});

rl.on('line', (line) => {
  // 1. 여기서 모든 라인은 독립적인 JSON 봉투여야 함
  const envelope = JSON.parse(line); 
  
  // 2. type에 따라 분기 처리 (Routing)
  if (envelope.type === 'token') {
    webContents.send('render-token', envelope.data.text);
  } else if (envelope.type === 'tool_call') {
    webContents.send('show-overlay', envelope.data);
  }
});

```

### 요약하자면

* **Ndjson**: 한 번에 다 받지 않고, **한 줄씩 실시간으로** 받아 처리하겠다.
* **봉투(Envelope)**: 데이터 내용물(Data)과 타입 정보(Type)를 **항상 함께** 보내서, 뜯어보기 전에도 무엇을 해야 할지(라우팅) 알겠다.

이 구조 덕분에 에이전트가 길게 생각할 때도 채팅창은 멈추지 않고 실시간으로 반응하며, 도구 실행 시에도 자연스럽게 UI 오버레이가 뜰 수 있는 것입니다. 이 개념을 적용하면 UI 개발팀이 CLI 출력 파싱 때문에 고생할 일이 크게 줄어듭니다.
