## 1. 폴더 및 모듈 관계 흐름

ReAct(Reasoning & Action) 패러다임은 에이전트가 스스로 **의사 결정을 위한 생각(Thought)**을 전개하고, 지정된 도구들을 통해 **행동(Action)**을 수행하며, 수집된 결과 및 예외 로그를 **관찰(Observation)**하고 피드백 루프를 순환하는 유기적 구조입니다. `omx` 시스템 내에서 이 흐름은 다음 모듈들을 통해 물리적인 아키텍처로 조립되어 작동합니다.

```
+─────────────────────────── Thought (추론 레이어) ───────────────────────────+
│ - model_reasoning_effort 규격을 통한 고연산 추론 모드 결정                  │
│ - StreamParser를 활용한 type: "token", subType: "reasoning" 실시간 필터링    │
+─────────────────────────────────────┬───────────────────────────────────────+
                                      │
                                      ▼
+─────────────────────────── Action (행동 레이어) ────────────────────────────+
│ - prepareAutoresearchRuntime(): Git Worktree 생성 및 행동 영역 격리         │
│ - candidate.json 및 candidate_commit 생성으로 물리적 변경사항 아티팩트화     │
+─────────────────────────────────────┬───────────────────────────────────────+
                                      │
                                      ▼
+────────────────────────── Observation (관찰 레이어) ─────────────────────────+
│ - runAutoresearchEvaluator(): 외부 검증 명령 실행 결과 파싱/평가             │
│ - results.tsv & latest-evaluator.json 상태 물리 원고 기록                   │
│ - CompletionAudit를 통한 체크리스트 증적 검증                                │
+─────────────────────────────────────┬───────────────────────────────────────+
                                      │
                                      ▼
+─────────────────────── ReAct 반복 루프 (The Loop) ──────────────────────────+
│ - AutoresearchLedgerEntry 일지에 관찰 결과 누적                             │
│ - 차세대 이터레이션에서 Ledger 기록 기반 지시서(instructions.md) 동적 갱신   │
+─────────────────────────────────────────────────────────────────────────────+
```

## 2. ReAct 기법 모듈 역할 개요

| ReAct 단계 | 핵심 기술 및 구현체 | 핵심 역할 및 사양 |
|---|---|---|
| **Thought (사고/추론)** | `model_reasoning_effort` & [Electron/src/main/cli/stream-parser.ts](Electron/src/main/cli/stream-parser.ts) | 고정밀 추론 연산 세기(low, medium, high, xhigh) 적용 및 SSE 스트림 내의 추론 전용 토큰 실시간 추출 콜백 구현 |
| **Action (행동/실행)** | `Git Worktree` 격리 & [src/autoresearch/runtime.ts](src/autoresearch/runtime.ts) | 호스트 안전성 확보를 위한 격리 워크트리 및 에이전트 수정 산출물(`candidate.json`) 생성 및 커밋 핸들링 |
| **Observation (관찰/검증)** | `Evaluator` 스크립트 실행 & [src/ralph/completion-audit.ts](src/ralph/completion-audit.ts) | 컴파일러 오류, 단위 테스트 지표(`score`), 보안 한계선 위반 여부를 계량 분석하여 상태 파일과 원장에 즉각 갱신 |
| **Loop (반복/개선)** | `Autoresearch Ledger` & `instructions.md` 조립 | 실패 및 성공 관찰 피드백 데이터를 누적하여 차기 이터레이션의 인스트럭션 기획 시 프롬프트 컨텍스트로 제공 |

## 3. 세부 단계별 구현체 정밀 분석

### 3.1 Thought (사고 및 추론) 아키텍처

에이전트가 어설픈 단순 치환을 지양하고 깊고 폭넓은 지식 구조를 자아낸 후 행동에 나서도록 시스템 매개변수를 제어하고, 그 사고 과정을 실시간으로 격리 수집합니다.

#### A. 에이전트 추론 세기 제어 (`reasoningEffort`)
*   [한글해석본/코드분석/dependency-map-use-codex.md](한글해석본/코드분석/dependency-map-use-codex.md#L274)를 참조하면, 루트 설정 파일 조립 시 에이전트별 `model_reasoning_effort` 키에 따라 추론 노력을 차별 가중할 수 있게 설계되었습니다.
*   [Electron/src/test/EL-211.test.ts](Electron/src/test/EL-211.test.ts#L86-L101)와 [Electron/src/test/EL-212.test.ts](Electron/src/test/EL-212.test.ts#L141-L148)에서 볼 수 있듯이, 모델 질의 옵션에 `reasoningEffort: "high"` 또는 `"xhigh"` 조건이 부여될 때 CLI 단에 `--high` 및 `--xhigh` 옵션을 변환 인계하여 AI 모델 내부에서 깊은 생각 과정(Thought)을 활성화하도록 유도합니다.

#### B. SSE 스트림 내 Thought(추론 분리) 식별과 시각화
*   에이전트가 생각 중인 내용을 일반 출력 텍스트와 혼선 없이 시각화하기 위하여 전용 스트림 파서가 개발되었습니다.
*   [Electron/src/main/cli/stream-parser.ts](Electron/src/main/cli/stream-parser.ts) 모듈 및 [Electron/src/test/stream-parser.test.ts](Electron/src/test/stream-parser.test.ts#L123-L129)를 들여다보면 SSE 데이터 처리 중 `{ type: "token", subType: "reasoning" }` 포맷을 실시간 식별하는 분기가 수립되어 있습니다.
*   파서는 이 전용 토큰을 감지하면 즉시 `onThinkingToken` 콜백을 화이어하여, 사용자 대시보드 및 IPC 버스에 에이전트의 세부 추론 내용만을 단독 렌더링되게 격리 제어합니다.

---

### 3.2 Action (행동 및 변화 유도) 아키텍처

Thought 가설이 성립한 직후 에이전트는 독립된 작업 샌드박스를 조종하여 실제 코드를 변경하고 커밋하는 등 적극적인 행동을 취합니다.

#### A. 가상 샌드박스 개화 및 Git Worktree 조율
*   자유도 높은 코드 교정과 리팩토링 과정이 호스트 기반 소스 코드를 훼손하지 않아야 하므로 물리 격리된 영역을 수립합니다.
*   [src/autoresearch/runtime.ts](src/autoresearch/runtime.ts)의 `prepareAutoresearchRuntime` 함수는 에이전트가 실시간 행동하는 시점에 메인 리포지토리 루트와 대조할 수 있는 전용 Git Worktree 디렉토리를 구축해 격리 안정을 달성합니다.

#### B. 아티팩트 구조화 (`candidate.json`)
*   임의 파일 수정 행위 외에도, 에이전트의 Action 결과를 추적하기 위해 구체적인 후보 객체를 명세화합니다.
*   `AutoresearchCandidateArtifact` 명세에 맞추어 `candidate.json`에 자신의 Action 완료 상태(`status`), 조정한 대상 커밋(`candidate_commit`), 상세 Rationale 변이 사조(`description`, `notes`)를 가시적으로 발행합니다.

---

### 3.3 Observation (관찰 및 검증 피드백) 아키텍처

행동에 대한 물리적인 인과론적 반응을 측정하여 성공 및 개선 지점을 과학적(점수, 통과율, 빌드 정상화 등)으로 도출합니다.

#### A. 자동 연산 기전 측정 (`Evaluator`)
*   [src/autoresearch/runtime.ts](src/autoresearch/runtime.ts)에는 외부 검증 도구로부터 피드백을 환류하기 위해 `AutoresearchEvaluationRecord` 정보가 운영됩니다.
*   `evaluator.command` 호출을 트리거하여 컴파일 오류 코드, 이터레이션 대비 연동 소스 코드의 `score` 변이, `stdout` 및 `stderr` 로그의 오류 및 예외 스택 데이터를 계량 보존하여 `results.tsv` 와 `latest-evaluator.json`을 통해 기록 관찰을 실현합니다.

#### B. 정적 요건 및 완료 요건 가치 측정 (`CompletionAudit`)
*   [src/ralph/completion-audit.ts](src/ralph/completion-audit.ts)에 설계된 `evaluateRalphCompletionAuditEvidence` 모듈은 에이전트가 제시한 Rationale, 변경 증적, 준수 체크리스트들을 규칙 엔진 및 파일 시스템 위상과 병합 대조하여, 완성도 합격(`complete: true`) 여부나 거절 원인(Reason, Source)을 종합 산정합니다.

---

### 3.4 ReAct 피드백 반복 루프 구조화

`omx`는 Thought -> Action -> Observation에서 생성된 피드백 증적을 버리지 않고 순차 전송하여 자율 개선 속도를 보장합니다.

1.  **원장 누적**: 에이전트의 매 시도마다 `AutoresearchLedgerEntry` 오브젝트에 이터레이션 고유 평가 정보와 Decision 상태(`keep` | `discard` | `ambiguous`) 및 실패 예외를 축적 기록합니다.
2.  **지시서 재생성**: 매 신규 회차 진입 직전 `buildAutoresearchInstructions`가 발동되어 수립된 원장 정보를 이터레이션 메타 요약본(`AutoresearchInstructionLedgerSummary`)으로 재조립합니다.
3.  **지침 연동**: 새 지침서 `instructions.md` 파일에 이전 이터레이션 행동(Action)들이 어떤 스코어 변동(Observation)을 유도했는지 텍스트로 보태어 줍니다. 이를 다음 생각(Thought) 컨텍스트로 학습한 에이전트는 결함을 정밀 피하고 올바른 대안을 마련하여 목표 스코어를 신속 확정하게 됩니다.

## 4. 프롬프트 내의 ReAct 통합 기제

`omx` 프로젝트 전체 프롬프트 역시 물리 엔진의 ReAct 흐름과 모순이 없도록 각 행동 주체별 역할을 설계 정책으로 동조합니다.

*   **[한글해석본/prompts/executor-ko.md](한글해석본/prompts/executor-ko.md)**: 무모한 코드 변경(Action)을 단행하기 전에 전후 문맥을 해독한 독점적 Rationale 증적 서술을 엄격 요구하여 Thought의 구조성을 단장합니다.
*   **[한글해석본/prompts/designer-ko.md](한글해석본/prompts/designer-ko.md)**: `package.json` 등을 통한 환경 감지 기전을 사전에 설계하여, 올바른 환경 관찰(Observation) 후에 그에 정렬된 프레임워크 최적 기조(Action)가 표출되도록 프롬프트 가이드를 통합 매핑했습니다.

결절적으로 `omx` 솔루션은 고차원 논리 추론력을 지닌 거대 언어 모델이 추상 세계에 미치지 않고 물리적 환경 변화와 컴파일러 신호와 결탁하도록 조형하는 업계 최고 준위의 ReAct 에이전트 구현 교과서를 제시하고 있습니다.
