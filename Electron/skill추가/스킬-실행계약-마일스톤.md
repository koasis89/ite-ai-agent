# 스킬 실행 계약 레이어 구현 마일스톤

작성일: 2026-07-03
기준 문서: Electron/skill추가/전체-스킬-실행계약-설계안.md

## 목표

docx 단일 기능 중심 구현을 전체 스킬 공통 실행 계약 체계로 확장한다.
완료 기준은 다음과 같다.

- 모든 skill 항목이 상태 정책과 계약 레이어에 등록됨
- canonical, merged, deprecated, internal 처리 일관성 확보
- Electron UI에서 정책 기반 표시와 실행 가능 범위가 명확함

## M1. 코어 계약 기반 구축

범위

- core 계약 타입 정의
- 표준 에러 코드와 오류 봉투
- status-policy 모듈
- registry 인터페이스

산출물

- src/skills/contracts/core.ts
- src/skills/errors/codes.ts
- src/skills/errors/mappers.ts
- src/skills/policy/status-policy.ts
- src/skills/executor/types.ts

완료 기준

- active, deprecated, merged, alias, internal 정책 함수 단위 테스트 통과
- input/output 검증 헬퍼 테스트 통과

리스크

- status 정의와 manifest 상태값 불일치

대응

- manifest 파서에서 미정의 status 감지 시 빌드 실패 처리

## M2. docx 파일럿 이전


범위

- 기존 md-docx 경로를 SkillActionContract 기반으로 래핑
- execute 흐름에 input/output 검증 삽입
- fallback 저장 시 표준 오류 봉투 적용

산출물

- src/skills/contracts/actions/docx.ts
- src/skills/executor/docx-executor.ts
- src/skills/executor/dispatch.ts
- Electron IPC 연결 업데이트

완료 기준

- 기존 docx변환 UI 동작 유지
- 실패 케이스가 표준 코드로 반환
- 회귀 테스트 통과

리스크

- 기존 export-ipc와 책임 경계 충돌

대응

- actionId 기준 라우팅 단일화

## M3. execution/planning 스킬 확장



범위

- execution/planning 카테고리 스킬 계약 등록
- canonical 라우팅 적용
- merged/alias 처리 로직 확정

산출물

- contracts/actions 하위 카테고리 분할
- registry 자동 등록 로더
- canonical resolver

완료 기준

- execution/planning 전체 action 라우팅 성공
- merged/alias 호출 시 canonical 변환 검증 통과

리스크

- 카탈로그 변경 시 등록 누락

대응

- manifest diff 기반 누락 검사 CI 추가

## M4. shortcut/utility 및 UI 정책 반영



범위

- shortcut/utility 카테고리 정책 반영
- internal UI 숨김
- deprecated 호출 경고 UX 추가
- 스킬 패널에 계약 상태 표시

산출물

- Electron renderer 스킬 패널 업데이트
- policy banner 컴포넌트
- preload API 확장

완료 기준

- UI에서 internal 비노출
- deprecated는 기본 차단, 고급 옵션에서만 허용
- merged/alias 안내 문구 정상 노출

리스크

- 사용자 혼란 증가

대응

- 상태별 도움말 툴팁 제공

## M5. 운영 안정화 및 품질 게이트 완성



범위

- 전체 회귀 테스트
- 로그/감사/메트릭 도입
- 문서화 및 릴리스 체크리스트 완성

산출물

- 통합 테스트 스위트
- requestId 기반 실행 로그
- 운영 문서 및 장애 대응 가이드

완료 기준

- 주요 경로 E2E 통과
- 상태별 정책 위반 0건
- 릴리스 리뷰 승인

리스크

- 테스트 비용 증가

대응

- 카테고리별 스모크 테스트와 전체 야간 회귀 분리

## 트랙별 병렬 작업 권장

- 트랙 A
  - 계약 타입, 정책, registry
- 트랙 B
  - Electron IPC, Preload, Renderer 연동
- 트랙 C
  - 테스트 및 운영 로그

## 테스트 계획

단위 테스트

- input schema reject
- output schema reject
- status-policy transition
- canonical/merged/alias 라우팅

통합 테스트

- Renderer -> Preload -> Main -> Executor
- docx 파일 저장 fallback
- deprecated 호출 차단

회귀 테스트

- 기존 chat 응답 출력
- 기존 export 경로
- 기존 skill list 조회

## 완료 선언 기준

- manifest skill 전체가 registry에 반영됨
- 상태 정책 위반 호출이 표준 코드로 반환됨
- Electron UI와 실행 정책이 동일한 소스 오브 트루스를 사용함
- 릴리스 체크리스트 전 항목 완료
