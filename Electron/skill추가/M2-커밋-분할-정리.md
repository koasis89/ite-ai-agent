# M2 커밋 분할 정리

작성일: 2026-07-03
기준: M2 구현 결과 (docx 계약 래핑 + IPC 연결 + 회귀 테스트)

## 목적

M2 결과를 리뷰/롤백/추적이 쉬운 3개 커밋으로 분리한다.

- Commit A: 코어 이관
- Commit B: IPC 연결
- Commit C: 테스트

참고: 빌드 산출물 파일(dist-electron)은 커밋 대상에서 제외 권장.

## Commit A. 코어 이관

### 범위

docx 기능을 스킬 실행 계약 레이어로 이관한다.

### 포함 파일

- src/skills/contracts/actions/docx.ts
- src/skills/executor/docx-executor.ts
- src/skills/executor/registry.ts
- src/skills/executor/dispatch.ts

### 스테이징 예시

```powershell
git add src/skills/contracts/actions/docx.ts
git add src/skills/executor/docx-executor.ts
git add src/skills/executor/registry.ts
git add src/skills/executor/dispatch.ts
```

Constraint: 기존 UI 채널/호출 방식 회귀 금지

## 커밋 제외 권장

아래 파일은 로컬 번들 검증 산출물이라 기본 제외 권장.

- dist-electron/main.cjs
- dist-electron/main.cjs.map

필요 시 제외 명령:

```powershell
git restore --staged dist-electron/main.cjs dist-electron/main.cjs.map 2>$null
```

## 검증 순서 권장

1. Commit A 스테이징 후 `npx tsc --noEmit`
2. Commit B 스테이징 후 Electron main 번들 확인
3. Commit C 스테이징 후 `node dist/scripts/run-test-files.js dist/skills/__tests__`
4. 마지막으로 `git status --short`로 커밋 분리 상태 확인
