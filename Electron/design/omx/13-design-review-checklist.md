# 13장 설계 검토 체크리스트

## 0) 사용 방법
- 대상: Electron/design/omx 하위 모듈 상세 설계 문서
- 목적: 문서 완성도와 구현 가능성, 운영 리스크를 사전에 검증
- 판정: 각 항목을 Pass, Partial, Fail 중 하나로 기록

## 1) 계약 누락 점검
- [ ] 외부 인터페이스가 요청, 응답, 이벤트 구독 기준으로 분리되어 있는가
- [ ] 필수 필드와 선택 필드가 명시되어 있는가
- [ ] 실패 응답 형식과 오류 코드 규약이 정의되어 있는가
- [ ] preload, ipc, service 간 채널명 또는 함수명 매핑이 일치하는가

## 2) 상태 전이 일관성 점검
- [ ] 상태 전이 경로가 시작, 진행, 종료, 취소, 실패를 포함하는가
- [ ] 금지 전이 또는 예외 전이가 문서화되어 있는가
- [ ] 불변식이 최소 3개 이상 정의되어 있는가
- [ ] 전이 시 부수효과가 명시되어 있는가

## 3) 오류 시나리오 점검
- [ ] 입력 검증 오류와 런타임 오류가 구분되어 있는가
- [ ] 복구 경로가 사용자 관점에서 설명되어 있는가
- [ ] 실패가 다른 파이프라인에 전파되지 않도록 격리 전략이 있는가
- [ ] 재시도 가능 여부 또는 중단 조건이 정의되어 있는가

## 4) 테스트 커버리지 점검
- [ ] 단위 테스트 항목이 핵심 규칙을 직접 검증하는가
- [ ] 통합 테스트 항목이 주요 시퀀스 end-to-end를 포함하는가
- [ ] 회귀 테스트 항목이 과거 결함 재발 방지에 연결되는가
- [ ] 테스트 미대상 영역과 사유가 기록되어 있는가

## 5) 운영 리스크 점검
- [ ] 성능 병목 가능 지점과 완화 전략이 있는가
- [ ] 보안 리스크(민감정보, 권한, 입력 오염)가 식별되어 있는가
- [ ] 모니터링 또는 로그 가시성 항목이 정의되어 있는가
- [ ] 오픈 이슈의 우선순위와 담당자 지정 항목이 존재하는가

## 6) 종합 판정

- 상태 전이 일관성: [ ] Pass [ ] Partial [ ] Fail
- 오류 시나리오: [ ] Pass [ ] Partial [ ] Fail
- 테스트 커버리지: [ ] Pass [ ] Partial [ ] Fail
- 운영 리스크: [ ] Pass [ ] Partial [ ] Fail
