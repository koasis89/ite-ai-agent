## 🎫 EL-224. 복합 상태(team + ralph 등) 전이 규칙 파서 및 롤백 방지 로직 구현

- **우선순위**: P0
- **실행 순서**: 20번째
- **전제 티켓**: Phase 2의 상태 엔진 (EL-218)
- **그룹**: Logic (State Hardening)
- **목표**: 지연된 스킬 발생 시와 상태 정합성 오류 시 사용자에게 클로드 스타일의 비간섭적 알림을 제공한다.
- **대상 경로**:
  - `src/main/state/multi-state-validator.ts` (신규)
  - `src/renderer/components/DeferredSkillsNotice.tsx` (신규)
- **핵심 구현 로직 (UI 특화)**:
  1. **지연 스킬 플로팅 토스트(Toast)**: 주 스킬 실행으로 인해 `ralph`와 같은 서브 스킬이 지연(Deferred)된 경우, 메인 뷰를 가리는 모달 대신 **우측 하단에 조그만 플로팅 토스트(Toast)** 형태로 "⏳ 다음 작업 대기 중: ralph" 메시지를 노출시켜 시각적 피로도를 낮춤.
  2. **비정상 롤백 차단**: 워크플로우가 `in_progress` 단계에 진입한 후 임의 조작이나 오류로 `planning` 단계로의 상태 파일 덮어쓰기가 감지되면 무효화(Reject) 처리.
  3. **명시적 상태 초기화(Clear) 가이드 배너**: 불법적인 롤백 시도가 감지될 경우, 채팅창 최상단에 주의(Warning) 색상의 얇은 인라인 배너를 띄워 "상태 충돌이 감지되었습니다. 여기를 클릭하여 상태를 초기화(omx state clear)하십시오." 형태의 Actionable UI 제공.
- **DoD (완료 기준)**:
  - 지연 스킬 알림이 플로팅 토스트 형태로 우측 하단에 표시됨.
  - 잘못된 상태 전이 시도 시 채팅창 상단에 상태 클리어 액션 인라인 배너가 표시됨.
- **체크리스트**:
  - [x] 지연(Deferred) 스킬 정보 노출용 우측 하단 플로팅 토스트 컴포넌트 추가
  - [x] 롤백 룰 위반 탐지 시 채팅창 상단 인라인 경고 배너 렌더링
  - [x] 경고 배너 내 상태 초기화 명령 트리거 액션 버튼 연동
  - [x] `EL-224.test.ts` (복합 상태의 Deferred 분리 및 금지된 전이 튕겨내기 테스트)

## 🎫 EL-225. 로그-카탈로그 정합성(Drift) 실시간 검증 모듈 및 리포팅 구축

- **우선순위**: P1
- **실행 순서**: 21번째 (마지막 대미 장식)
- **전제 티켓**: EL-207 (훅 로그 테일러)
- **그룹**: Debug/Ops (Consistency)
- **목표**: 런타임에 쌓이는 플러그인 디스패치 로그(`.omx/logs/hooks-*.jsonl`)와 매니페스트 원본(`manifest.json`)을 비교하여 설정 불일치(Drift)를 실시간으로 탐지하고 복구 경로를 제공한다.
- **대상 경로**:
  - `src/main/ops/drift-detector.ts` (신규)
  - `src/main/ipc/ops-ipc.ts` (신규)
- **핵심 구현 로직 (보완됨)**:
  1. **실시간 교차 검증**: 로그 테일러를 통해 인입되는 훅 이벤트의 `source` 및 `context` 식별자와 시스템에 로드된 `manifest.json`의 플러그인 목록을 비교하고, 미등록 섀도우 호출을 탐지.
  2. **오차율(Drift Rate) 산출**: 전체 도구 호출 대비 실패/미등록 도구 호출 비율을 산출해 상태 대시보드의 System Health 지표로 전송.
  3. **실행 가능한 복구 경로 제공 (추가)**: 정합성 위기(예: Drift Rate 10% 초과) 시 UI 글로벌 배너에 경고와 함께 환경 동기화(`omx setup` 재실행) 버튼을 노출해 원본 카탈로그와 로컬 환경 재동기화 액션 제공.
- **DoD (완료 기준)**:
  - 플러그인 또는 설정 파일 오염 시 UI 상단에 정합성 위기 알림이 노출됨.
  - 사용자에게 실행 가능한 복구 버튼이 제공되고 재동기화 흐름이 동작함.
  - 빌드/검증 타임에도 동일한 로직으로 무결성 검사 수행 가능.
- **체크리스트**:
  - [x] 로그-매니페스트 교차 검증기(Drift Detector) 로직 작성
  - [x] **[옵스(Ops) 게이트]** 정합성 위기 임계치 초과 시 알림 및 자동 복구(`omx setup` 트리거) 버튼 연동
  - [x] 빌드/검증 타임 CI 환경에서도 동작 가능한 Headless 검사 인터페이스 개방
  - [x] `drift-detector.test.ts` (미등록 플러그인 호출 모킹 시 Drift 감지 및 알림 트리거 검증 완료)
